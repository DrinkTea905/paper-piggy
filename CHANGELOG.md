# 更新日志

本文件记录用户可见的变更。格式参考 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，
版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

> 维护约定：**每次发版必须更新本文件**（见 [docs/MAINTENANCE.md](docs/MAINTENANCE.md) 的 checklist）。
> 版本号的唯一事实源是 `config.APP_VERSION`。

## [1.0.33] — 2026-07-20

PaperPiggy 新增明亮与夜间双主题，并把界面进一步收束为统一的暖白纸张层级：常用操作、卡片、
圆角、阴影和间距采用同一套视觉规则。新手指引与 Agent 使用指引也移到更容易找到、又不挤占
内容的位置；综述的生成与更新统一交给 Agent 完成。

> **升级方式**：本版是界面与交互更新，不需要重建索引。v1.0.32 用户可直接使用应用内更新；
> 也可以运行完整安装器覆盖原位置，已有文献、索引、模型、综述和 Agent 文件不会被删除。

### 新增

- **明亮 / 夜间模式** —— 左上角提供太阳与月亮切换，应用会记住选择；明亮模式保留原来的纸张小猪，
  夜间模式切换为战斗小猪，并采用真正的深色内容层级。
- **小猪头像成为帮助入口** —— 点击左上角小猪即可随时打开新手指引；头像增加常驻书本角标，
  首次使用会显示一次“点小猪，随时打开新手指引”的定位提示。

### 改进

- **统一桌面应用风格** —— 保留深灰窗口框架，内容区采用暖白纸张层级，主操作使用珊瑚粉强调；
  卡片、按钮、圆角、阴影与间距统一，夜间模式和窄窗口同步适配。
- **文献与综述工具区更紧凑** —— 宽窗口尽量保持单行，避免后方留下大块空白；窄窗口改为容器内
  横向滚动，不把整页撑宽，也不隐藏任何筛选或操作。
- **Agent 页信息顺序更清楚** —— 深梅红横幅依次显示“AI 助手驱动你的知识库”、
  “本地库服务在线”和“Agent 使用指引”；指引直接合并进横幅，不再额外占用一整条内容区。
- **综述库更专注于阅读与管理** —— 保留按研究主题分类、检索、筛选、核验和维护工具，去掉冗长说明；
  综述的写作、更新与重新生成统一由 Agent 完成。

### 调整

- 移除网页端“生成综述”“重新生成”入口及对应公开接口，避免网页与 Agent 两套写作流程并存。
- 删除库总览中原有的大幅新手指引卡片，释放首屏空间；完整指引内容仍保留在应用内。

### 改进

- **窄窗口同步适配** —— 综述工具区在较窄窗口自动分行，筛选与操作仍保持完整，不产生横向溢出。

  图标同时交给标题栏和任务栏后放大。

## [1.0.30] — 2026-07-19

PaperPiggy 首次引入新的小猪图标方案；应用顶栏与内容框架重新收束，综述库改成按研究主题整理的书架，
页面多时也能按主题、标题和状态快速找到并舒适阅读。

> **升级方式**：本版不需要重建索引。v1.0.29 用户可直接使用应用内更新；
> 旧综述会按其来源与现有 AI 主题的重合度自动归类，不调用大模型、不消耗 API 额度，也不会改写正文。

### 新增

- **综述主题书架** —— 左侧集中显示研究主题和页数，右侧用紧凑目录呈现综述；支持标题搜索、
  类型筛选、未核验筛选，以及按更新时间、标题或来源数量排序。
- **自动归类 + 手动整理** —— 来源相近的综述自动归入现有本地 AI 主题；也可新建自定义主题、
  固定单页、重命名或删除主题。删除主题只会让其中页面恢复自动归类，不会删除任何综述。
- **页内阅读目录** —— 较长综述会根据二、三级标题生成页内目录；正文采用更合适的阅读宽度、
  字号和行距，主题、来源、核验、历史与原文入口仍然完整保留。

### 改进

- **应用框架更像桌面软件** —— 保留稳定的 Windows 原生窗口边框，重做品牌区、导航胶囊、
  激活状态、阴影与内容容器层级；深色主题和窄窗口同步适配。
- **综述列表更安静** —— 删除操作收进每页的“更多”菜单，列表优先展示标题、主题、来源数量、
  日期和核验状态，避免一屏堆满大卡片与危险按钮。
- **新图标接入应用界面** —— 网页品牌位使用新的小猪图标，窗口侧的本地 `.ico` 缓存可随源图更新重建；
  Windows 快捷方式、安装器和多尺寸图标的完整收尾见 1.0.31。

## [1.0.29] — 2026-07-19

综合推荐经过38条真实查询校准，不再让来源档次轻易压过正文相关性；综述库也会在写入前拦住
不存在的文献编号，并找出旧页面中的错误来源和重复标题。

> **升级方式**：本版不需要重建索引。v1.0.28 用户可直接使用应用内更新；
> 更新后可在“综述库 → 体检”查看旧页面的来源错误和重复标题，并交给 Agent 按清单修复。

### 改进

- **综合推荐有真实标尺** —— 建立30条专题查询与8条离题查询的金标集，以25条调参、13条盲测。
  API 检索的来源评价加成由 `0.50` 调整为 `0.30`：调参组前三命中率与前八召回率均为100%；
  盲测组保持两项100%，同时比旧权重取得更高的排序质量。本地检索分数尺度不同，保留原值，不生搬硬套。
- **错一位的来源不再污染综述** —— 新写或修改综合页时，来源 key 必须真实存在；发现错误会整页拒绝、
  列出近似候选，不会只保存一半。对话命中既有综合页时，会展开成其原始论文来源。
- **综述体检新增两类问题** —— 可列出“来源指向不存在的文献”和“重复标题/研究问题”；
  只报告旧问题，不静默改写用户已有内容。
- **摘要名称不再含混** —— Agent 查看单篇文献时，明确分开 Zotero/题录摘要与用于语义检索的 SAC 摘要，
  避免把正常的检索摘要误判成错误。
- **检索摘要质量门槛更完整** —— 新增相邻中文短语重复和异常连续标点检查，明显损坏的生成结果不会进入索引。

### 修复

- 新建综合页时，如果 Agent 把页面标题或同一研究问题也放进正文开头，应用会精确去掉重复外壳；
  正文中正常的小标题不受影响。

## [1.0.26] — 2026-07-19

库总览重新编排为更紧凑的文库概况：评价分布与最近入库合并展示，文献构成不再堆叠说明和箭头；
评价规则会直接列出用户改过的项目，并可逐项或一次性恢复出厂设置。

> **升级方式**：本版没有新增运行组件，也不要求清空或重建索引。v1.0.25 用户可直接使用应用内更新；
> 若更新后看到“题录分类规则需要更新”，只需运行一次“手动更新知识库”，无需重新深索或重建语义索引。

### 改进

- **文库概况更集中** —— 四档评价与最近入库进入同一张概况卡片，不再用“全库四档评价”等重复标题；
  学科与更新提示也收进卡片标题区，减少总览的纵向占用。
- **文献构成更易浏览** —— 操作说明与标题同排，客观标签移除装饰性箭头；默认先展示主要项目，
  其余项目按需展开，点击标签仍可直接查看对应文献。
- **自定义规则看得见、退得回** —— 折叠的“评价规则”显示实际改动数；展开后置顶列出每项的
  当前值与出厂默认值，可恢复单项或恢复全部。把某项改回默认值后不再留下冗余自定义记录。
- **规则修改立即生效** —— 目录/文献性质映射调整会即时更新评价分布、浏览和检索排序，
  不触碰客观目录数据，也不要求重建索引。
- **升级提示更准确** —— 只有题录分类指纹变化时明确提示运行一次手动更新，并说明无需深索或重建；
  正常的运行环境、深索和语义索引状态不再挤进警告区域。

## [1.0.24] — 2026-07-19

PaperPiggy 现在用同一套评价覆盖期刊论文、书籍、书章、学位论文、法源、案例、标准、报告、
数据集、预印本、会议论文和网页，不再让非期刊材料落进一套不适用的期刊六档。

> **升级方式**：本版没有新增运行组件，也不要求清空或重建索引。v1.0.23 用户可直接使用应用内更新；
> 也可运行完整安装器覆盖原位置，文献索引、模型、综述库、手动改档和 Agent 文件都会保留。

### 新增

- **全类型文献统一评价** —— 普通页面只显示一个客观标签，如“三大刊”“CLSCI”“SSCI Q1”
  “TSSCI”“书籍”“博士论文”或“法源”；主观评价统一为权威、顶级、核心、普通四档。
- **真实文献性质筛选** —— 浏览页和 MCP 都可按十二类真实性质过滤，并与收藏、主题、深索状态、
  排序和分页叠加；网页白皮书、报纸中的正式司法意见会按内容性质归类。
- **目录与性质映射可调整** —— 库总览集中展示四档比例、内部权重、客观标签/性质明细、目录来源、
  上游版本和半年检查状态；用户可只调整当前学科的映射，不改写客观目录数据。
- **正式 TSSCI 与 JCR 目录** —— 正式 TSSCI 法律学门与个人台湾偏好表分离；SSCI/JCR 目录更新为
  ShowJCR JCR2025 快照，并记录来源提交和检查日期。
- **娱乐显示名** —— 新增“法学（开发者增强：夯到拉）”，完全复用开发者增强规则、权重、排序、

## [1.0.6] — 2026-07-15

修复深索时应用崩溃 + 大量文献被误判为「扫描件」——根因是 PDF 提取库的线程不安全。

### 修复
- **深索时整个应用窗口崩退**：PDF 提取用 4 线程并发调 PDFium，而 **PDFium 不是线程安全的**，
  多线程同时访问会损坏其内部堆（Windows 事件日志确诊：故障模块 `pdfium.dll`，
  异常码 `0xc0000374` 堆损坏 / `0x80000003`）。改为**所有 PDF 提取全局串行**，绝不并发。
  实测：同一批 500 个 PDF，并发时约 70% 失败/偶发崩溃，串行后 100% 成功、不崩。
- **大量有文字的文献被误标「🚫 扫描件·需 OCR」**：正是上面并发竞态导致的随机
  「Failed to load」（同一 PDF 单独打开明明好好的），被误判成扫描件、深索篇数虚低。
  串行修复后，重跑「深索全部」会自动重试这些篇并成功入库。
- PDF 提取加瞬时失败重试（文件被 OneDrive / 杀软 / Zotero 短暂占用时多试两次）。

> 升级后请重新点一次「深索全部」：之前因崩溃/误判没入库的文献会自动补上（不会重复劳动，已成功的跳过）。

## [1.0.5] — 2026-07-15

修复 API 模式下批量深索/建索引的崩溃、假完成与卡死。

### 修复
- **API 模式深索全库会崩**：嵌入调用没有兜底，一次 API 失败（余额不足→403、限流、网络超时）
  就会带 traceback 掀翻整个后台任务。现在：致命错（密钥/余额/模型名）立即优雅中止并给人话提示；
  限流/网络类跳过该篇、连续多次才判停；**任一篇没嵌成功都不会谎报完成**，已嵌的部分照常保存、可续跑。

- **进度条卡死在「正在提升检索质量 X/Y」**：语义嵌入中途失败后，进度条会永远停在那儿谎报进行中。
  现在区分「真在跑」（动进度条）与「上次未完成」（静态提示「点『手动更新知识库』继续」）。

> **本文件是什么**：在《实现设计_综合层与答案沉淀.md》（把"检索到的理解"沉淀成带引用的 wiki 页 + MCP 读—综合—写回闭环）之上，回答下一个问题——**综合层建好之后，让 LocalKB 长出哪些 agent 工作流、怎么排期、每条怎么接到现有检索+wiki层+MCP 上**。
> **怎么用**：本文是"选什么、按什么顺序做"的决策蓝图；每选定一条工作流，再据其卡片写独立的实现设计文档（如《实现设计_引注核验.md》），交给新对话实现。
> **依据**：对 `LocalKB\app\*.py` 全量代码核对（file:line 均在正文标注）+ 14 路多智能体联网调研（2026 法律 AI 与学术研究 AI 的 SOTA/同类产品、诚信与法律实证、落地接法），逐条带取舍。调研覆盖 Harvey / CoCounsel / Lexis+AI / Vincent / Clearbrief / BriefCatch、Elicit / PaperQA2(FutureHouse) / Undermind / STORM·Co-STORM / Ai2 ScholarQA / NotebookLM / Deep Research、以及北大法宝·法宝AI+ / 通义法睿 / 幂律 等国内法律大模型。
> **引注体例基准**：《法学引注手册》（2019，中国法学会法学期刊研究会推荐，何海波执笔；已出第二版）。

## 0. 一句话与总纲

**LocalKB 要长成的，是"半自动研究助手 + 引注质检器"，不是"自动写论文机器"。**

综合层已经把范式从"每次重推"挪到"可累积"。下一步不是把 agent 放开去成文，而是**沿"读—综合—写回"闭环，把学者写作的每一环都做成"带源初稿 + 强制核验 + 人类负全责"的助手**。这一条定位贯穿全部 11 条工作流，不可动摇——调研里所有 SOTA（OpenAI/Google Deep Research、STORM/Co-STORM、PaperQA2、CoCounsel）与国内权威建议（最高检"权威内部语料库 + RAG + 多模型交叉验证"）指向同一结论：**底物没错，价值在编排与人工闸，不在再造检索。** 而 LocalKB 的底物（bge-m3 稠密 + BM25 + RRF + reranker + 期刊分层 + 页级引用 + wiki 综合层 + MCP 写回）恰好是这套 SOTA 公认的正确地基。

**三条不可逾越的红线（触及每一条工作流，见 §7 详述）：**

1. **幻觉引注 / 伪造判例。** 2026 全球已编录 1300+ 起向法院提交 AI 伪造判例/引注的案件，制裁从 2023 年 5000 美元升到 2025 年 5.5 万美元；国内四款主流法律 AI 平均幻觉率 30.8%；北京通州法院 2025-12 一案入选人民法院案例库，明确"提交未经甄别核实的 AI 虚假案例应担责，情节较重参照《民事诉讼法》第 114 条"。**每一条引注必须能页级回溯到 `data/extracted` 的真实页文本，回不去的不许进终稿。**
2. **未披露 AI 即学术不端。** 《学术出版中 AIGC 使用边界指南 2.0》（2024.9）、中科院《规范使用 AI 技术诚信提醒》（2024.9）、复旦《本科毕业论文 AI 使用规定》（2024.11）、《学位法》（2025.1.1 施行，§37 伪造/剽窃可撤销学位）——共同底线：AIGC 直接生成的文字必须可追溯并披露（工具名+版本、参与阶段、贡献度、核验方式），否则视为抄袭。
3. **CNKI AIGC 检测。** 知网检测 3.0 抓的是"逻辑连贯但认知特征缺失"，同义替换只能降 10–15%、突破不了阈值；多校（人大、西南大学等）将 AIGC 检测纳入毕业审查，法学期刊跟进，AI 率红线约 20–30%。**一键平滑成文反而更像 AI——agent 只做"带源初稿 + 核验"，终稿由学者改写。**

**一个反直觉但最重要的判断：11 条工作流不是 11 个独立工程，它们共享约 6 个新件（§3 地基层）。其中两个——页级引文核验器（G1）与《法学引注手册》规则引擎（G2）——同时是旗舰"写论文工作流"的核验脊柱。因此最高杠杆的第一步不是"写论文"整条，而是先把 G1/G2 这两个核验原语建出来。**

## 1. 三条横切铁律（约束每一条工作流）

这三条是 2026 SOTA 的共识，也是把"帮到学者"与"给学者埋雷"分开的分水岭。任何工作流的实现都必须遵守。

### 铁律一：裁判-召回解耦（verifier ≠ generator）
核验必须由**独立的裁判器**（独立 prompt / 独立模型）完成，不能让生成答案的同一模型自评——否则会自证幻觉（Mata v. Avianca 的教训：问 ChatGPT"这案子真的吗"它会说真的）。SOTA 依据：FACTS Grounding、CiteTracer、ProvenanceGuard、RARR 的 Agreement Gate。**落到 LocalKB**：便宜 key（DeepSeek/硅基 `L.chat_once`，llm.py:32）跑批量初判，疑难/高危句经 MCP 交前沿模型（Claude Code）复核；核验取证一律走 `W._gather_evidence`（wiki_store.py:256，已自动剔除旧 wiki 行），保证"不拿 AI 综合去核验 AI 引注"。

### 铁律二：规则做格式、模型做实质
《法学引注手册》里**确定性的**部分（字段顺序、标点、〔〕vs（）、"载"的触发、页码/案号格式）用规则引擎，几乎 100% 可判且可解释，绝不交给 LLM 自由发挥（BriefCatch 路线）。LLM 只做**语义判断**（引文是否被来源支持、命题一致性）。**关键例外——引领词"参见/见/又见/另见/转引自"的选择必须留人工**：它是对"作者是否核实了原文"的实质声明（"转引自"=未核实原文只是转引，"参见"=已核实概括引用），工具擅自把"转引自"改成"参见"等于替作者虚假声明，属学术不端而非格式问题。

### 铁律三：三态而非二值（库覆盖有限，"查无 ≠ 幻觉"）
LocalKB 只有 ~2000 篇，不是法学全集。任何涉及"库里有没有"的判定，一律用**三态**：`库内可核验 / 库内不符(mismatch) / 库外无法核验(not_in_lib)`——只有"解析到库内某文献但页码或命题不符"才标 mismatch，"库里没查到"只降级为"待人工核"，绝不判成幻觉。依据：朴素 cite-check 的假阳性率实测可达 65.9%（arXiv 2606.21155）；把真实的库外引用冤枉成幻觉，反噬学者信任、动摇整个工具的可信度。这与设计文档 §8"拒绝把 LLM 判定当断言落盘"同源。

## 2. 分类法与工作流全景

用户给的四型（**调动 / 生成 / 追踪 / 核验**）恰好覆盖学者写作的全生命周期。下表是 11 条候选的鸟瞰（详细卡片见 §4，优先级与排期见 §5–6）。

| # | 工作流 | 主类型 | 一句话价值 | 结论 |
|---|---|---|---|---|
| 4.1 | **写论文工作流（旗舰）** | 生成·核验（复合） | 研究问题→检索→提纲→分节起草(绑源)→引注核验→定稿的端到端编排 | **该做**（做成半自动助手，非成文机；分波建） |
| 4.2 | 反向溯源 / grounding 核验 | 核验 | 给一句话/一个论断，反查库内出处、判四态、给页码 | **该做（第一优先）**——底物契合最高、边际成本最低、是 G1 原语 |
| 4.3 | 引注核验与规范化 | 核验 | 给带脚注的稿子，逐条对库核验(真实性/页码/命题一致)+按手册格式化 | **该做（旗舰级优先）**——差异化杀手锏 |
| 4.4 | 专题 / 文献综述 | 生成 | 就一个研究问题产出结构化、页级引用、期刊加权的综述初稿 | **该做**——深化现有 `synthesize_topic` |
| 4.5 | 文献卡片 / 读书笔记 | 生成 | 单篇文献→结构化卡片(命题/方法/证据/关系)，Zettelkasten 互链 | **该做**——低风险沉淀层，日常高频 |
| 4.6 | 观点谱系 / 学说争议地图 | 生成·调动 | 就某争点梳理各家立场、期刊加权排布、画争点树 | **该做**——高学术价值、差异化；但立场误植风险高 |
| 4.7 | 观点压力测试 / 审稿辅助 | 核验 | 用库内对立文献红队论点/初稿，模拟审稿人诘问 | **该做**——投稿前自我红队；复用 G1 |
| 4.8 | 投稿匹配 / 期刊选择 | 调动 | 给稿子，按主题契合 × 期刊层级推荐目标期刊 | **该做**——近零新增代码的快赢 |
| 4.9 | 增量文献监测 + stale 重生 | 追踪 | 新文入库→圈出受影响 wiki 页→标 stale→生成 delta 摘要 | **该做**——基础设施价值；对接现有 stale 机制 |
| 4.10 | 选题 / 研究缺口发现 | 调动·追踪 | 扫全库找欠研究议题、悬而未决争点 | **该做（缓）**——价值高但"虚假空白"风险高，做 co-pilot |
| 4.11 | 矛盾候选提示 | 核验·追踪 | 入库时检出新文与既有高权威文献的语义对立，只读提示 | **缓做**——风险最高（法学"不同教义框架≠矛盾"） |

每张卡片八字段：**定位 / 类型 / 诚信与法律风险 / 复用现成件 / 流水线阶段+人工闸 / 触及的法学红线 / 分期 / token 成本**。旗舰在前。

### 4.1 写论文工作流（旗舰） 🚩

**定位（一句话价值）**：把"研究问题→文献扫描→提纲→分节起草(带页级引用)→引注规范化→自我核验"串成一条**带多道人工闸**的编排，让学者在"研究助手 + 起草助手"上省力，而非把论文交给机器写。

**类型**：生成·核验（复合，串起 4.2/4.3/4.4）。

**诚信与法律风险**：
- 〔high〕**幻觉引注/伪造判例**——无源起草必编引注（SciSpace 被大量投诉编造不存在的期刊/作者；国内法律 AI 幻觉率 30.8%）。缓解=分节起草强制绑 `data/extracted` 真实页文本，回不去的不许落。
- 〔high〕**未披露即抄袭 + 触 CNKI 检测**——一键平滑成文正是被判高 AI 率的特征。缓解=agent 只出"带源初稿+核验"，保留人工在提纲/起草/核验处的真实介入痕迹，G4 导出披露声明。
- 〔med〕**citation faithfulness**——模型先写结论再贴勉强沾边的来源，页码对但论点非出自该页；法学对"参见/见/转引自"极敏感。缓解=G1 逐条 quote 比对。
- 〔med〕**越冷门越幻觉**——前沿/小众议题（学者最想写、也最危险）虚构率最高。缓解=三态如实报"库内无支持"。

**复用哪些现成件**：`W._gather_evidence`(wiki_store.py:256) 建证据池 → `R.search`(retriever.py:385) 分节取证 → `journal_grading.resolve_journal_weight`(resolver.py:80) 加权援引 → `R._page_cite`(retriever.py:166) 页级引用 → `W.synthesize_topic`/`save_answer`(wiki_store.py:325/221) 承载综述/提纲中间产物 → G1 核验、G2 格式化、G4 披露 → MCP 6 工具（Claude Code 经 `search_localkb`/`get_wiki_page`/`save_synthesis` 编排）。**架构要点：这条工作流最佳载体是 Claude Code（前沿模型）驱动现有 MCP 读工具 + 本地写草稿文件，而非服务端廉价 key**——设计文档 §6.3 已论证"综合维护者是前沿模型时质量最好"。

**流水线阶段 + 每阶段人工闸**（★=硬闸）：
0. **意图澄清**（人工）：agent 回问 2–3 个澄清问题（学科/管辖/时间范围/要综述还是论证）→ ★人确认研究问题与边界。零成本、杠杆最高（对标 Deep Research 的 Clarification、Co-STORM 的人类注入）。
1. **规划**（廉价 key）：拆 3–6 个子探索方向 + 检索计划（plan-and-execute）。
2. **迭代检索**（复用 R.search）：每方向 2–3 轮"检索→看证据→改写 query→再检索"，设轮数硬预算防跑飞 → ★人筛掉无关/低质/幻觉来源、定纳入清单。
3. **提纲（最硬人工闸）**：仿 Co-STORM——先建 grounded 证据池，agent 生成提纲、每个节点挂已召回的 `key+页码` → **★人确认提纲结构与论证主线**（STORM/Deep Research 都在动笔前锁定提纲；这是学者认知贡献的核心，绝不交 agent）。
4. **分节起草**（可选前沿模型经 MCP）：每段调 R.search 取证，只允许引用真实页文本，每句主张回填 `key+页码`，禁止无出处生成。
5. **引注规范化**：走 4.3（G2 格式化 + 三态核验）。
6. **自我核验（独立一遍）**：走 4.2/4.7——独立裁判器逐条把引注拉回原文页比对，标 unsupported/页码错位/张冠李戴 → **★人工终审，逐条核对每一条判例/法条/页码**。
7. （可选）**投稿匹配**：走 4.8。

**触及的法学红线**：伪造判例/幻觉引注（红线1）；未披露/AI 率超阈（红线2/3）；"转引自"误用（G2 铁律二）；把 LLM 判定的"矛盾/通说"当断言落盘（禁止）；学者须为核心智力贡献负责（提纲/论证主线不得由 agent 主导）。

**分期**：
- **MVP**：阶段 0→2→3→4→6 的最小闭环，作为一个 Claude Code 技能（编排现有 MCP + G1），产物=带源初稿 + 核验报告的本地 markdown。**不新增服务端代码即可跑起来**（依赖 G1 就绪）。
- **最高杠杆一步**：**阶段 3 的"grounded 提纲人工闸"**——它同时防跑偏、防越界代写、防 CNKI 检测，是全流程价值枢纽。

**token 成本**：单篇全流程（前沿模型）≈ 50 万–150 万 token（综述 1–2 次 + 提纲 1 次 + 分节 5–8 节各 1–2 次 + 核验 + 红队）；重活（规划/初筛/格式）改用廉价 key 可压到几十万级。检索/重排本地近免费。**铁律：每次 LLM 综合都对应一次用户显式请求，never eager。**

### 4.2 反向溯源 / grounding 核验（= 地基件 G1 的产品形态）

**定位**：给一句话或一个论断，反查库内最可能的出处，逐句判**四态**（支持/部分支持/不支持/无法判定）并给证据 + 页码；本质是"检验/裁判"而非"生成"，天然是写论文工作流的"落笔前/投稿前"闸门。

**类型**：核验。

**诚信与法律风险**：
- 〔high〕**裁判器谄媚/确认偏误**——论断含错误前提时，检索与 judge 倾向找语义相近证据来"附和"而非证伪（用户想核验、系统却背书）。缓解=铁律一解耦 + 主动追加"该论点反例/被批评/被推翻"的反向 query（类比 Shepard's/KeyCite）。
- 〔high〕**伪造判例**——把"库里查到相似案"误报为"支持"等于给伪造背书。缓解=判例/法条援引单列最严核验 + 强制人工闸；"库内无=显式报缺"，绝不"找个最像的凑"。
- 〔med〕**"无法判定"被悄悄归入支持/不支持**——必须显式暴露 uncertain 并计入人工闸，不能二值化。
- 〔med〕**绿灯疲劳**——高通过率诱导跳过复核（Stanford 实测 Lexis 17%/Westlaw 33% 仍幻觉）。缓解=默认展开高危项、对"支持"也留一键跳原文、统计人工复核率。

**复用现成件**：`R.search`（判例/法条援引偏 BM25 词法精确匹配，学理观点偏 bge-m3 稠密，RRF 正好覆盖两种模式）→ `reranker.scores` 精排取证据页 → `data/extracted` 页文本 + `official_pages` → `W._gather_evidence`(自动剔旧 wiki 行防自我循环) → 分层 LLM（廉价 key 批量四态初判，前沿模型经 MCP 复核疑难）→ `save_answer`/MCP `save_synthesis` 存核验页(标 by_agent)。SOTA 锚点：FActScore/SAFE 原子分解、FACTS Grounding 独立裁判、SemanticCite（小模型四态 84% 加权准确）、Scite Smart Citations。

**流水线阶段 + 人工闸**：①论断分解成原子命题（区分 事实/判例援引/法条援引/学理观点 四类，核验策略不同）→ ②反向检索候选出处（R.search）→ ③reranker 精排选证据页 → ④**独立裁判**（与召回解耦，逐命题判四态 + 抽支撑句+页码）→ ⑤聚合报告（句级红绿高亮 + 置信度 + `key+页码`）→ ⑥人工确认后写回。★闸(a)：判"不支持/无法判定"的高危句强制人看原文；★闸(b)：判例/法条援引即便判"支持"也建议人工复核；★闸(c)：写回只给人（MCP 不给删）。

**触及的法学红线**：伪造判例/幻觉引注（红线1）；"转引自"陷阱（库内文献本身可能是二手转引，溯源只到该文献会把二手当一手）；未披露（定位须限死"溯源核验/引注核对"这类校对性用途，可披露可留痕）。

**分期**：
- **MVP**：单句四态核验，输出 `{态, 证据句, key, official_pages, 置信度}`；先做"存在性 + quote 比对 + 语义支撑分"，暂不做原子分解。
- **最高杠杆一步**：把它做成 MCP 只读工具 `verify_citation`，让 Claude Code 在写作时**选中一句→反查→四态报告→高危句人工看原文→确认后回写**，形成"落笔即核验"闭环。

**token 成本**：每条论断 1 次检索(免费) + 1 次裁判 ≈ 3–8k token；一节 20 条主张 ≈ 60–150k，廉价 key 完全可承受。

### 4.3 引注核验与规范化（《法学引注手册》）

**定位**：给一份带脚注引注的中文法学稿件，逐条对库核验（存在性/页码正确性/**命题一致性**）、按手册格式化引领词与体例、标疑似幻觉引注，产出可回溯的核验报告并可写回 wiki。这是防御性 + 规范性双目标工作流，也是 LocalKB 相对云端 cite-check 的**差异化杀手锏**——云端工具只对判例库，LocalKB 有中文期刊论文的**页级全文**，能做"命题级核验"（抓 name/cite mismatch）。

**类型**：核验。

**诚信与法律风险**：
- 〔high〕**name/cite mismatch 漏检**——最危险的不是假案例，而是真实来源被断章/反向援引（HAQQ 实验：每个链接都能打开、表面全对，读原文才发现讲的是别的事）。缓解=命题级一致性核验，不止存在性；纯语义相似度会容忍实体替换（当事人/日期/条号被换仍判高分），需加轻量结构校验。
- 〔high〕**把"库中查无"判成幻觉并落盘**——违背铁律三，冤枉库外真实引用。
- 〔med〕**引领词误用改变学术含义**——把作者本应标"转引自"的改成"参见"=替作者虚假声明已核实原文（铁律二）。
- 〔low〕**页码错位**——`official_pages` 官方页码 vs PDF 物理页偏移、跨页命题、脚注串页 → 用 `official_pages` 映射 + 邻页窗口容错。

**复用现成件**：G2 规则引擎（体例格式化）+ G1 核验器（存在性/quote/命题一致三层）；`R.search` 作检索落地引擎（用 作者+标题+被引命题 构造 query）；`reranker.scores` 做 Clearbrief 式 semantic score；`journal_grading` 作分诊（tier 低/needsReview 的引注升级人工复核优先级）；`W._gather_evidence` 取证并剔旧 wiki 行；`save_answer` 存核验报告为 kind=cite-report 页。SOTA 锚点：CoCounsel/KeyCite 的 name/cite match、Clearbrief 的事实-来源逐句对齐、BriefCatch 的"规则做格式"、CiteTracer 多智能体（抽取→检索→核验→判定，边界样本转人工）。

**流水线阶段 + 人工闸**：①引注抽取（从脚注解析结构化单元：作者/题名/出处/页码/引领词/引文文本）→ ②分类（itemtype 决定套哪套模板）→ **★闸1·格式层·可全自动放行**：规则校验字段顺序/标点/〔〕/"载"/编号，生成 diff → **★闸2·真实性层**：R.search 落地判存在，查无只标"待核"（三态）→ **★闸3·quote+pinpoint 层**：页级文本逐字比对 + reranker 判命题是否落在所标页 → **★闸4·实质层·强制人判**："参见/见"选择、是否真支持论点、转引合规性，工具只给证据与建议、作者签字定稿 → 报告输出（每条引注五态：格式/存在性/引文准确/页码/支持度 + 证据链 + 建议，可留痕审计）。

**触及的法学红线**：name/cite mismatch（红线1 的隐蔽变体）；"转引自/参见"实质声明（铁律二，工具不得自动改）；伪造判例进诉讼文书（对"拟用于诉讼/仲裁"的稿件须显著提示"AI 核验不免除独立核实义务"，援北京通州法院案例）；未披露（报告内置披露声明模板）。

**分期**：
- **MVP**：格式层（G2 规则 diff）+ 存在性三态 + quote 字符级比对；命题一致性先只对"疑似不符"的少数条目上语义裁判。
- **最高杠杆一步**：**命题级一致性核验（name/cite mismatch）**——这是攻主流产品最难点、且只有本地全文库能做的差异化。

**token 成本**：单篇稿件（~30 脚注）：抽取 + 逐条检索(免费) + quote 比对(免费) + 仅对可疑条目上语义裁判 ≈ 10 万–30 万 token；可批量、可分层（廉价 key 抽取粗筛，前沿模型只判命题一致）。

### 4.4 专题 / 文献综述

**定位**：给一个法学研究问题，产出结构化、页级可回溯引用、按期刊层级加权的**综述初稿**——是"人监督下的证据合成脚手架"，不是"自动作者"。深化现有 `synthesize_concept/synthesize_topic`。

**类型**：生成。

**诚信与法律风险**：
- 〔high〕**伪造/幻觉引注**——实证仅 64–68% 的 LLM 生成引用对应真实论文，联网也消不掉；2025 顶会论文 2.59% 含幻觉引文（较 2024 涨 10 倍）。缓解=页级引用必须回链 `data/extracted`，回不去的 claim 不落盘。
- 〔high〕**越界代写**——"AI 独立完成文献综述"在中文学术规范下普遍被认定为需研究者批判判断的核心任务、AI 替代即不端。定位必须钉死为"初稿脚手架 + 人改写"，UI/文案不得暗示可直接采用。
- 〔med〕**popularity bias**——模型系统性偏向高被引/主流观点，埋没少数说与新近成果，综述会失衡。缓解=按期刊层级分组呈现观点分布。
- 〔med〕**期刊权重被误用为"观点正确性"**——层级只反映来源可信度，不等于论点更对。

**复用现成件**：`W.synthesize_topic`(wiki_store.py:325) 作骨架，深化为"分节 serial 合成"；`W._gather_evidence` + `R.search` 加**多轮迭代召回**（借 Undermind/PaperQA2 Citation Traversal：首轮召回后抽高频关键词/被引 key 做二三轮扩展）；`journal_grading` 参与 RRF 后重排 + 分组呈现；`build_ai_topics.py` 的 24 簇作主题骨架；G1 核验闸 + G2 脚注 + G4 披露。SOTA 锚点：PaperQA2 的 **RCS（Reranking + Contextual Summarization：每段先压成带出处 summary 再合成）**、STORM 的 grounded citation graph（检索期即绑引用不事后补）、Ai2 ScholarQA 的逐节 serial + quote 级归因、Co-STORM 的人介入 curation。

**流水线阶段 + 人工闸**：①问题澄清/范围界定 → ★人确认研究问题、时间范围、纳入排除标准 → ②多轮检索召回（R.search + 引文扩展）→ ★人审检索式与召回清单 → ③筛选（宁滥勿缺，Elicit 式把 AI 当"第二审稿人"）→ ★人删假阳性（agent 只读+人删权限的天然落点）→ ④逐段 **RCS** 压成带 `key+页码` 的 summary（不可回链页码的直接丢）→ ⑤生成大纲 → **★人改大纲再写正文**（STORM/Co-STORM 核心介入点）→ ⑥分节 serial 合成，每节只引本节已分配的带页码证据 → **★引用核验闸**（每条脚注回链页文本比对，不过则红标"待核"）→ ⑦持久化（frontmatter 记 sources/generated_by/stale）。

**触及的法学红线**：幻觉引注（红线1）；越界代写（红线2）；把"矛盾/共识"当断言落盘（禁止，只作"分歧提示"块并列展示双方 `key+页`）；AI 率超阈（红线3，产物是初稿需人重写）。

**分期**：
- **MVP**：在现有 `synthesize_topic` 上加 RCS（逐段带出处压缩）+ 分节 serial + 硬核验闸；先单轮召回。
- **最高杠杆一步**：**RCS + 每条 claim 强制页码回链**——从架构上把幻觉引注按住。

**token 成本**：单主题综述 ≈ 15 万–40 万 token（多轮检索免费 + 逐段 RCS 每段 ~1k + 分节生成）；缓存命中后 0 成本，stale 后按需重生。

### 4.5 文献卡片 / 结构化读书笔记

**定位**：给单篇文献生成结构化卡片（核心命题/方法/证据/局限/与他篇关系），Zettelkasten 式互链；AI 只降低建卡与互链成本，命题真值与关系落盘决定权留给学者。低风险、日常高频的沉淀层。

**类型**：生成。

**诚信与法律风险**：
- 〔high〕**幻觉引注落入卡片并被带进论文**——卡片"证据/关系"字段若由 LLM 自由生成而非严格锚定原文页，即量产"可信度伪装"的假引注。缓解=每字段强制携带 `key+official_pages`，LLM 只做压缩改写不做事实新增（对齐 NotebookLM 段落级 grounding）。
- 〔high〕**AI 生成的"卡片间关系/矛盾"被当学术断言**——语义连接常是似是而非的伪关系（false links）。缓解=互链只出"候选"标 `needs_confirm`，人点确认才转正。
- 〔med〕**过度简化/漏关键**——AI 卡片倾向测琐碎细节、漏核心论证框架，错误时用词更自信。
- 〔low〕**期刊分层被误用为"命题正确性"背书**——须与真值判断严格解耦。

**复用现成件**：新增 **kind=card**（G3），沿用现成 YAML frontmatter，进同一 LanceDB 表 `row_type="wiki"` 即可被检索（零新增存储层）；`W._gather_evidence` 做"单篇内取证 + 跨篇找关系候选"；`data/extracted` 页文本作证据源；`journal_grading` 给卡片"证据强度"打标（非真值）；`synthesize_concept` 把跨篇同名概念聚合成概念页（hub-and-spoke）；MCP `save_synthesis`(标 by_agent) 写回、删只给人。SOTA 锚点：NotebookLM 段落级 inline 引注、Scholarcy 字段化抽取、Atlas/Saner 的"AI 建链候选 + 人确认"。

**流水线阶段 + 人工闸**：①摄入解析（`data/extracted` 已就绪）→ ②字段化抽取（LLM 从单篇抽 命题/方法/证据/局限）→ **★闸A：卡片草稿人工审校**（尤其法条/判例/引注字段）→ ③卡片渲染（填模板 + G2 脚注）→ ④互链（R.search 找候选相关卡，写 `links` 标 `needs_confirm`）→ **★闸B：人确认关系成立**（AI 只给候选，不自动落断言）→ ⑤持久化（standby，标 by_agent）→ ★闸C：删只给人。

**触及的法学红线**：判例/案号/法条字段严禁 LLM 自由生成（红线1，必须锚原文页或权威库）；把 LLM 判定的关系/矛盾当断言（禁止）；AI 率（卡片文本进正文前须人工改写消化）；未披露（frontmatter 生成披露清单）。

**分期**：
- **MVP**：单篇触发 → hub 卡（总览）+ N 张原子命题卡，每字段绑 `key+页码`，互链只出候选。
- **最高杠杆一步**：**卡片即 kind=card 的 wiki 页**——复用整条检索/嵌入/降权/披露管线，卡片天然可被 R.search 检索到，与综述/概念页互链成图。

**token 成本**：单篇 1–2 次调用 ≈ 2 万–4 万 token；抽取用廉价 key，关系判定/终审用前沿模型。

### 4.6 观点谱系 / 学说争议地图

**定位**：就某法学概念或争点（如"数据财产权归属""个人信息侵权过错认定""AIGC 可版权性"），从库内召回讨论段落→抽取"谁主张什么立场"→归并成若干对立学说（肯定说/否定说/折中说）→按期刊权威度与学者代表性加权排布，输出可回溯到页码的"学说地图"。服务写论文的"文献综述/学说梳理"一节，是差异化空白位。

**类型**：生成·调动。

**诚信与法律风险**：
- 〔high〕**立场误植（false attribution）**——系统说"某学者主张否定说"实际其持折中说或相反，是硬伤级学术错误，被引学者当面证伪损信誉；LLM 会复制训练数据里既有的错误归属。缓解=立场抽取后、归并前设**必设人工闸**核对归属。
- 〔high〕**幻觉引注/伪造页码**——综述"文字"仍是 LLM 生成，措辞可能悄悄越出证据。缓解=图约束生成（Falkor-IRAC 思路：每句只能引图中已抽到的节点/边）。
- 〔med〕**"转引自"红线**——从 A 文献读到"B 教授认为…"直接标成引 B 原文=制造隐性转引/断章（铁律二）。
- 〔med〕**"通说/少数说"被权重算法误导**——用期刊层级+论文数排出的"通说"≠学理通说（Consensus meter 的教训：数论文≠真共识，高引老文可能已被推翻）。
- 〔med〕**库覆盖偏差**——某说关键代表作不在库/无 PDF，地图系统性漏掉一派却给"完整谱系"错觉。

**复用现成件**：`W._gather_evidence`(内含 R.search + 剔旧 wiki 行)，topk 调大 + 多轮补召确保少数说不漏；立场抽取用 MCP 让前沿模型逐段判 `{key, 页码, 立场标签, claim 摘录, premise}`，页码/原文取自 `data/extracted` 天然页级回溯；`journal_grading.resolve_journal_weight` 给各"说"打连续权重，渲染时按 weight 排布节点大小/高度、needsReview 标灰——**这正是对 Consensus meter"不加权"缺陷的直接修正，是 LocalKB 独有卖点**；`synthesize_concept`/`save_answer` 落成 kind=debate 页，`links` 连相关 concept 页；`ai_topics.json`/收藏夹树限定争点边界，减少跨领域同名概念污染（如"过错"在侵权法 vs 刑法）。SOTA 锚点：STORM 的视角引导提问、Typed Claim Network（带类型的支持/反驳/延伸边）、Stakeholder Suite 的"行动者-议题-论点"三元。

**流水线阶段 + 人工闸**：①争点界定/query 规范化 → ②证据召回（多轮补召）→ ③立场抽取（逐段判该文献持何立场）→ **★闸1（必设）：核对"系统说 X 文献持否定说"是否属实**（防 false attribution）→ ④归并聚类（同质立场合成"说"）→ ⑤关系建图（support 边可自动落，**attack/矛盾边强制人工确认后才写**，★闸2）→ ⑥加权排布 + 成稿 → ★闸3（建议）：每条引注人工抽验真页真话。

**触及的法学红线**：立场误植（学术硬伤）；"转引自"（铁律二）；把 LLM 判定的矛盾/反驳当断言落盘（禁止）；伪造判例（涉实务观点引指导案例时严禁伪造案号）；AI 率 + 未披露（成品附"AI 贡献声明 + AI 率自查"页）。

**分期**：
- **MVP**：单争点的"立场表"（各文献 `key+页码+立场标签+claim 摘录`）+ 期刊加权排序，**先不建 attack 边**（只列各说，不断言谁驳谁）。
- **最高杠杆一步**：**期刊分层加权的立场可视化**——底物已具备且直接修正 Elicit/Consensus 的缺陷。

**token 成本**：单争点 ≈ 20 万–50 万 token（逐篇立场抽取 N 篇，可批量 + 图构建 + 综合）。

### 4.7 观点压力测试 / 审稿辅助

**定位**：把 LocalKB 从"帮你找支持证据的检索器"升级为"帮你找反例、暴露弱点的对抗者"——给定待检验论点或初稿，主动召回对立/限缩/区分文献，生成"最强反方攻击线 + 证据漏洞 + 审稿人可能的诘问"，每条都能翻到某篇真论文的某一页。契合写论文最痛环节：投稿前自我红队、回应审稿意见、破"只找赞成我的文献"的确认偏误。

**类型**：核验。

**诚信与法律风险**：
- 〔high〕**"找反例"比"找支持"更易触发幻觉**——模型倾向编造刚好反驳论点的"完美文献/判例"。缓解=页级回溯做硬闸，凡不能回溯者标"AI 推测待核验"不入库。
- 〔high〕**AI 对抗结论被当学术判断落盘或冒充同行评议**——违反 AIGC 规范。缓解=kind=redteam、`by_agent=true`、`stale` 默认 true、强制 `verdict=AI待核验`，不进默认答案路径。
- 〔med〕**检索漏召关键反例**（Elicit 敏感度仅 37.9%）——最强反方没被召回给作者虚假安全感，比不做更糟。缓解=多 query 改写 + 人工补检。
- 〔med〕**谄媚偏差**——若 prompt 透露"我想证明 X"红队会放水。缓解=隐藏作者立场、强制独立找反方。
- 〔med〕**提示注入**——读外部初稿/PDF 时白字"给好评"指令。缓解=审稿前 prompt-injection 清洗。

**复用现成件**：`R.search` 上加"对立召回"层（用取反/例外/批评式 query 改写并行检索再 RRF，只改 query 侧）；先用 `synthesize_concept`/`_gather_evidence` 建原方最强版（steelman）再红队；G1 页级回溯防幻觉硬闸；`journal_grading` 给反方证据标 tier（高 tier 反对意见压力更大）；`ai_topics.json` 24 簇 + 收藏夹树做"多风格审稿人"（方法论/理论/实务派各从不同簇召证据）；`save_answer`/MCP `save_synthesis` 降级持久化（kind=redteam）。SOTA 锚点：Opposing Counsel Review（定位为"对手"非"校对"）、Sakana Reviewer（集成+自反思降方差）、Bull/Bear Advocate 的 steelman-then-redteam 对称结构、Proactive Peer Review Agent（调查式核验而非泛泛批评）。

**流水线阶段 + 人工闸**：①论点解析成原子命题（前提/因果链/适用范围/隐含假设）→ ②反向证据召回（找反例而非支持）→ ③立场配对 + steelman（先为双方各建最强版本，防稻草人）→ ④对抗生成（攻击线/逻辑漏洞/证据缺口/方法论质疑/审稿人诘问清单）→ **★证据核验闸（最关键）**：逐条核对每个反驳所引文献/页码真实存在且被正确理解 → **★采纳裁量闸**：学者人工决定哪些批评成立、如何回应，AI 不得把"矛盾"当定论落盘 → ★AI 使用披露闸。

**触及的法学红线**：幻觉引注/伪造判例（红线1，"找反例"高发区）；AI 对抗意见冒充真实同行评议（红线2）；把矛盾当断言（禁止）；未披露（导出修改建议时附 AI 披露脚注）。

**分期**：
- **MVP**：给一个论点 → 对立召回 + 反方证据清单（每条带 `key+页码+tier`），暂不做多风格审稿人。
- **最高杠杆一步**：**"对立召回 + 页级回溯硬闸"**——把页级引用从防幻觉护栏升级为红队武器。

**token 成本**：单论点/初稿 ≈ 10 万–30 万 token（query 改写初筛用廉价 key，终局对抗论证与核验用前沿模型）。

### 4.8 投稿匹配 / 期刊选择

**定位**：给一份稿件（题目+摘要+关键词+参考文献），按"主题契合度 × 期刊层级 × 作者约束"输出目标期刊排序清单 + 证据（该刊近年同主题代表作 + 页级引用），是决策辅助而非替作者投稿。**近零新增代码的快赢**。

**类型**：调动。

**诚信与法律风险**：
- 〔high〕**诱导一稿多投**——《著作权法》§32 给期刊 30 日等待期，但绝大多数刊投稿须知禁止并行投稿，坐实通常拒稿+拉黑+通报。缓解=强制 UI 文案"按拟投优先级逐一、遵守各刊等待期"，绝不鼓励群发。
- 〔high〕**幻觉外溢**——若复用同一 LLM 写投稿信/综述会带出伪造"该刊已发某文"或幻觉引注。缓解=凡事实句绑页级来源；本工作流**只出候选清单+证据+分层，绝不自动生成含事实承诺的投稿信、绝不自动投递**。
- 〔med〕**期刊分层口径不一致**——CLSCI（中国法学会，动态调整）/CSSCI（南大，来源版 vs 扩展版 vs 集刊）/北大核心/AMI 四套口径不同且逐年调整。缓解=`needsReview` 与目录版本号显式暴露。
- 〔low〕**本地语料非全量刊池**——冷门/新刊召不出，须明示"基于本地库、非完备推荐"。

**复用现成件**：`R.search`（把"摘要+关键词"当 query，召回最相似论文按 `journal` 字段聚合投票→候选期刊排序，正是 B!SON/Elsevier 同款"稿件 vs 各刊已发文"范式）；`journal_grading.resolve_journal_weight` 做"契合分 × 期刊层级"二维可解释排序 + `needsReview` 冒泡；参考文献命中本地库的 `key` 作引文耦合信号（EndNote 证实加参考文献提升 30% 准确率）；`data/extracted` 页文本 + `R._page_cite` 呈现代表作证据；`save_answer` 存 kind=submission 报告；可选 `synthesize_topic` 为 24 簇各建"该主题对口期刊画像"。SOTA 锚点：B!SON（Elasticsearch 词法 + 语义 embedding 双通道，与 LocalKB 同范式）、EndNote Manuscript Matcher、Elsevier Journal Finder（NLP + BM25）。**反直觉结论：多项研究中 BM25/TF-IDF 常持平或优于纯 embedding——支持保留词法通道 + RRF 融合。**

**流水线阶段 + 人工闸**：①输入（题目/摘要/关键词，进阶加参考文献）→ ②文本表示（BM25 + 语义）→ ③候选期刊召回（与各刊近年已发文聚合画像比相似度）→ ④重排打分 → ⑤结构化过滤加权（层级/OA/审稿周期/录用率/掠夺性黑名单）→ ⑥呈现候选 + 匹配证据 → **★人确认"主题契合"判断**（机器只给候选不拍板）→ **★needsReview 命中项人工核** → **★最终目标刊与投稿动作永远人来点**（系统绝不自动投递）。

**触及的法学红线**：一稿多投（学术不端）；幻觉"该刊已发文"（红线1）；期刊层级口径与作者单位考核不符（needsReview 透明）；评价指标（影响因子/录用率）不得 LLM 凭记忆填，须来自结构化字段或留空。

**分期**：
- **MVP**：R.search 聚合投票 → 候选刊 + `journal_grading` 分层 + 代表作证据，纯读、几乎零新增。
- **最高杠杆一步**：**"契合分 × 期刊层级"二维排序**，复用 `journal_grading` 现成引擎。

**token 成本**：检索(免费) + 1 次聚合/说明调用 ≈ 2 万–5 万 token。

### 4.9 增量文献监测 + stale 重生

**定位**：把 living review / continual-RAG 搬到本地——新条目入 Zotero 触发→判定命中哪些已存 wiki 页/主题簇→标 stale（只打旗不改正文）→生成"新增了什么"delta 摘要供人审。是"提醒+建议"系统，不是"自动改稿"系统。对接设计文档 Phase 3 的 stale 机制。

**类型**：追踪。

**诚信与法律风险**：
- 〔high〕**delta 摘要杜撰不存在的案号/法条/页码**——缓解=每条"新增点"挂新论文 `official_pages`，生成后自动校验页码存在且可回溯。
- 〔high〕**stale/delta 被当结论直接采信绕过逐查询检索**——违背设计哲学（wiki 非默认答案）。监测摘要是线索不是定论。
- 〔med〕**delta 越界到核心研究**（生成结论/分析而非仅整理线索）——国内规范限 AIGC 于检索/整理/润色等非核心环节。
- 〔med〕**"stale 疲劳"**——新文频繁触发满库 stale 旗，研究者麻木。缓解=按 tier 分级 + 周报式 digest 批量摘要，而非逐条打扰。
- 〔med〕**中文法学在 Semantic Scholar/OpenAlex 覆盖差**——纯靠国际 API 大量漏报，监测须以**本地 Zotero 新入库为主源**，外部 API 仅补充。

**复用现成件**：**G5 增量 diff 钩子**；触发层复用 Zotero 原生 feed（期刊/检索式 RSS→collection）；`localkb_build` 增量模式只对新 key 跑提取+入表（incremental indexing，不全量重建）；候选圈定用 `R.search`（新论文 title+abstract 当 query，命中的 `row_type="wiki"` 行即受影响页）；`ai_topics.json` 24 簇缩小 stale 候选范围；delta 生成走 **PaperWeaver 范式**（用 `_gather_evidence` 拉受影响页现有 sources 做上下文，让 LLM 只答"新论文相对这些已有证据新增/印证/冲突了什么"）；`journal_grading` 做优先级过滤（只对 CLSCI/CSSCI/台湾核心以上新文触发 stale）；`regenerate`(wiki_store.py:334) 人工触发重生；`index.json` 的 `by_source` 反查(wiki_store.py:94)。SOTA 锚点：Litmaps Monitor / ResearchRabbit（collection 即订阅）、Semantic Scholar Recommendations API、PaperWeaver、Cochrane Living Systematic Review 的"何时值得重生"判据。

**流水线阶段 + 人工闸**：①摄入触发（Zotero 新条目/feed 刷新，自动）→ ②候选圈定（新文向量化比对库内/wiki 向量，自动）→ ③影响面判定（命中哪些 wiki 页/AI 簇，阈值可调）→ **★闸1：人确认相关性**（滤掉误命中）→ ④stale 标记 + delta 生成（只写 frontmatter 的 stale/待办，不动正文）→ **★闸2：人决定是否 regenerate、新文是否进正文与脚注**。行业共识：②③④可自动，⑤⑥必须人工；"冲突/反驳"类判断绝不自动落断言。

**触及的法学红线**：delta 幻觉引注（红线1）；wiki 当默认答案（禁止）；把"冲突"当断言（禁止，只标 flag 引 Scite 式 support/contrast 交人判）；指导案例/司法解释须核现行有效性（是否被废止/修改）；未披露。

**分期**：
- **MVP**：G5 diff → 对每篇新文 R.search 命中 wiki 页 → 标 stale + candidate_keys（先不生成 delta 文字）。
- **最高杠杆一步**：**PaperWeaver 式 delta**（相对已有证据表述"新增/印证/冲突"，而非孤立介绍新论文）。

**token 成本**：每篇新文 检索(免费) + 每受影响页 delta 生成 ≈ 1 万–3 万 token/新文；周报式批量摊薄。

### 4.10 选题 / 研究缺口发现

**定位**：扫全库找欠研究议题、悬而未决争点、可切入的研究空白，产出候选选题清单 + 证据供学者人工裁决。做成"辅助发现 + 人工裁决"的 co-pilot，缓做任何自动落盘断言。

**类型**：调动·追踪。

**诚信与法律风险**：
- 〔high〕**"虚假空白"**——库仅 ~2000 篇的选择性收藏≠法学全域，把"本库未见"包装成"研究空白/首次提出"属对原创性的不实陈述。缓解=所有缺口措辞强制加检索范围与样本量限定词，禁绝"首次/学界空白/无人研究"等绝对表述，降级为"在本库 N 篇覆盖范围内、截至 {generated_at} 未见充分讨论"。
- 〔high〕**"看着新其实是改写抄袭"**——LLM 生成的"新颖"选题实为对已有观点换皮（arXiv 2502.16487《All That Glitters is Not Novel》）。缓解=novelty-checker 回检 + 必须向学者展示"最接近的已有文献"由人判断。
- 〔med〕**AIGC 未披露/超比例**；**把"矛盾/争点"当断言落盘**（禁止）；**期刊分层被误用为"重要性/真理性"背书**。

**复用现成件**：低密度区检测直接复用 `ai_topics.json` 24 簇 + 收藏夹树（统计每簇论文数/年份分布/期刊分层，稀疏簇、老化簇、高分层缺席簇即空白候选——纯统计 + 现有元数据 `year/journal_tier/collections`，不需新模型）；"局限/展望"段抽取用 `data/extracted`（定位"研究不足/尚待研究/有待进一步"段落，用 `official_pages` 锁页抽成对比矩阵——Elicit gap 范式的本地实现）；`_gather_evidence` + R.search 召证据，用召回收敛速度/新证据边际递减近似 Undermind 的 discovery curve（饱和度信号）；`reranker` 找"高相关但相互对立"对作争点候选；`synthesize_topic` 落 kind=gap 页且降级措辞；**MCP `save_synthesis` 前加 G1 引用核验闸**（每条 source 回查 `data/extracted` 真命中，未命中即拒写）。SOTA 锚点：Elicit gap（抽 future-work 成对比矩阵，不直接断言空白）、Undermind discovery curve、SciSpace Research Gap Finder。

**流水线阶段 + 人工闸**：①范围界定（学者给种子主题/收藏夹/簇）→ ②证据召回（混检+精排+分层召回相关论文及其"局限/展望"段）→ ③缺口候选抽取（对比矩阵/低密度区/争议对峙）→ ④novelty 回检（每个候选回检本库+必要时联网，标"疑似已被 X 文覆盖"）→ ⑤呈现落盘 → ★闸1：候选清单必经学者勾选，禁止自动扩散 → ★闸2："空白/矛盾/未被研究"措辞降级为可核证陈述 → ★闸3：写回前 G1 引用核验，不过不落盘。

**触及的法学红线**：虚假空白（原创性不实陈述）；伪新颖/改写抄袭；本工作流**不生成判例/指导性案例引用**（案例一律学者手工补，规避红线1）；把矛盾/空白当断言（禁止）；未披露。

**分期**：
- **MVP**：纯统计的"低密度区报告"（各簇论文数/年份/分层分布 + 稀疏簇标注），零 LLM、零幻觉。
- **最高杠杆一步**：**"局限/展望"段抽取成对比矩阵**（Elicit gap 本地实现，天然带页级证据）。

**token 成本**：统计(免费) + future-work 抽取（逐篇批量）+ novelty 回检 ≈ 10 万–30 万 token/次扫描。

### 4.11 矛盾候选提示（只读，风险最高）

**定位**：入库时对新文献核心 claim 检出与既有高权威文献的语义对立，作"未核实"只读提示。定位是"召回导向、只提不判、绝不落盘为断言"的辅助闸口，本质是 claim-level contradiction retrieval + 人工复核，**不是自动事实核查器**。

**类型**：核验·追踪。

**诚信与法律风险**：
- 〔high〕**把不同教义框架/学派之争误判为"矛盾"并断言（法学最核心误判）**——法学观点对立多源于解释论 vs 立法论、不同请求权基础、不同价值立场，是合法的 good-faith 分歧而非事实矛盾；ContraCrow/SciFact 的 NLI 范式为事实性文献设计，平移到规范性命题会系统性制造伪冲突；落盘为"A 被 B 驳倒"是对学者观点的失实断言，有名誉与学术评价风险。
- 〔high〕**幻觉/伪造对立文献、错配页码**——一旦附了错的对手方引注，危害大于不提示（LePhantomCite 证明 pincite/错引最难被自查）。
- 〔med〕**时间层/效力层混淆**——拿已修订法条、已被推翻判例去"反驳"现行观点（per incuriam）。
- 〔med〕**"矛盾提示即真理"锚定**——约 30% 的 ContraCrow 矛盾未获专家认可即为假阳；只读设计若 UI 暗示权威反诱导确认偏误。

**复用现成件**：候选召回复用 `R.search`（对新论文每条 claim 一次检索，RRF+reranker 拿语义近邻，几乎零新增基建）；用 `journal_grading`/`journal_tier` 把对手方**限定在高权威集合**（CLSCI/CSSCI/台湾核心），既符合"与既有高权威文献对立"设定又天然降噪；证据/判读锚定 `data/extracted` 页文本，矛盾提示必附双方 `key+页码`；分层 LLM（廉价 key 初筛，前沿模型复判）；`_gather_evidence` 只在原始论文行比（剔旧 wiki 行防回音室）；`ai_topics.json`/收藏夹树做**假冲突过滤**（同簇/同请求权基础才比，跨簇对立先降级为"议题相关"）；frontmatter 增 `contradiction_status: unverified` + `verified_by/verified_at`。SOTA 锚点：ContraCrow（三段式 + 11 点 Likert 对立强度 + 不落盘只作候选）、LegalWiz（四 agent：抽取→Context Analyzer→Judge→Confidence，明确保留人工闸）、SciFact 的三分类带弃权档（Supports/Refutes/**NEI 证据不足**）。

**流水线阶段 + 人工闸**：①claim 抽取（限定"可证伪的教义/事实命题"，剔除价值主张）→ ②定向召证（只在高权威集合找对手）→ ③对立判读（**三分类 Supports/Refutes/证据不足 + 连续对立强度**，必附双方页码，不用二元硬标签）→ ④假冲突过滤（Context Analyzer 检查不同教义框架/时间层/管辖/事实前提，命中则降级或弃权）→ ⑤只读呈现（标 stale/未核实，永不写成断言）。**★人工闸①**：低信度/证据不足自动不提示（精度置于召回之上）；**★人工闸②（落盘闸）**：默认根本不落盘，仅当人点确认才作为"学术争议"条目（写成"X 持…Y 持…存在分歧"而非"A 错 B 对"）落盘——**"矛盾确认"与"删除"同列为仅限人工的特权**。

**触及的法学红线**：把 LLM 判定的矛盾当断言落盘（设计文档 §8 明令禁止，本条最易复发）；不同教义框架被误判为矛盾（法学最核心误判）；幻觉对立引注（红线1）；效力/时间层混淆；未披露。

**分期**：
- **MVP**：**先只做只读候选**——`list_contradiction_candidates` MCP 只读工具（绝不加 auto-write 版），三分类带弃权 + 期刊分层收口 + 双方页码，暂不做自动判定与落盘。
- **最高杠杆一步**：**假冲突过滤器**（用 24 簇/请求权基础约束可比性），这是法学场景不翻车的关键。

**token 成本**：每篇新文 claim 抽取 + 逐 claim 检索(免费) + judge ≈ 3 万–8 万 token/篇；仅对高权威集合比，天然限流。

## 6. 建议建设顺序（三波次，衔接现有 Phase 0–3）

现有 Phase 0–3 已把综合层（wiki + MCP 读—综合—写回）建好。本蓝图是其上的"应用波次"。

### 波次一 · 核验地基 + 旗舰起步（最高杠杆，先建脊柱）
1. **G1 页级引文核验器**（=4.2 反向溯源 MVP）：`verify.py`，三态 + quote 比对 + 语义支撑分；暴露为 MCP 只读工具 `verify_citation`。
2. **G2《法学引注手册》规则引擎**：`cite_format.py`，itemtype 驱动模板 + 引领词只读建议。

4. **G4 披露导出器**：从 frontmatter 拼《生成式 AI 使用声明》。

### 波次二 · 生成与沉淀增厚（复用地基，扩面）

## 7. 合规与诚信总纲（每条工作流的通用底盘）

无论做哪条工作流，以下守则必须内建（不是可选项）。这也是把 LocalKB 与"给学者埋雷"的一键成文工具区分开的护城河。

### 7.1 三条红线的工程化守法（对应 §0）
- **红线1 幻觉引注/伪造判例** → **每条引注页级回溯到 `data/extracted` 真实页文本**（G1）；判例/法条/案号**不由任何生成类工作流产出**，一律学者手工补或走权威库另核；三态如实报"库内无支持"。
- **红线2 未披露即抄袭** → 每个 wiki 页强制 `generated_by/generated_at/by_agent/sources`（已有），G4 一键导出四要素披露声明（工具名+版本、参与阶段、贡献度、核验方式）；对接《学位法》§37、CAS《指南 2.0》、复旦/中科院规范。
- **红线3 CNKI 检测** → **agent 只出"带源初稿+核验"，终稿由学者改写**；保留提纲/起草/核验处的真实人工介入痕迹；不追求一键平滑成文。

### 7.2 人类负全责的四道通用闸
1. **提纲/论证主线闸**（生成类）：核心智力贡献必须人主导，agent 不得代定。
2. **写回前核验闸**（一切落盘）：G1 逐条页级回链，回不去的不落盘。
3. **实质判断闸**（引注/矛盾/立场）："参见/见/转引自"选择、"谁驳谁"、"某学者持何立场"——人裁决，agent 只给候选与证据。
4. **删除/采信闸**：agent 能读+综合+写回、**不能删**（MCP 设计已如此）；"矛盾确认"提升到与"删除"同级的人工特权。

### 7.3 明确不做（延续设计文档 §8 的拒绝清单）
- 不做全库 eager 预综合（token 埋葬点，一切按需+缓存）。
- 不把 LLM 判定的"矛盾/通说/共识"当断言落盘（法学最危险，本蓝图 4.6/4.11/4.9 反复守此线）。
- 不让 wiki 综合页替代逐查询检索（wiki 是附加缓存，主入口永远是 R.search）。
- 不做自进化写作规则、5 角色编辑部（对 solo 是仪式，且多 agent 辩论会放大偏差——EMNLP 2025）。
- 不做自动投递/自动写含事实承诺的投稿信（4.8）。
- 引领词、判例、"空白/首次"等绝对措辞不交给 LLM 自由发挥。

### 7.4 成本与模型档位诚实性
- 检索/重排本地近免费；LLM 成本按"每次综合对应一次用户显式请求"控制。
- **分层用模型**：重活（规划/初筛/抽取/格式）用用户自备廉价 key（DeepSeek/硅基 `L.chat_once`）；关键判断（命题一致性/立场归属/终局对抗/疑难核验）经 MCP 用前沿模型（Claude Code）——对抗评审里"廉价 key 综合质量不够"的顾虑在 agent 通道消失。
- 每页永远带 `generated_by` 戳 + 页级来源，让学者能审计、能一跳回原文；系统不假装综合层"可信到无需核对原文"。

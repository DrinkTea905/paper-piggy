# 新电脑从零试用 —— Windows 安装器验收清单

> 目的：在一台干净的 Windows 电脑上，从正式 Inno 安装器开始走完安装、首启、建库、检索、深索和应用内更新，找出只在开发机成立的问题。版本号以 `src/config.py` 的 `APP_VERSION` 为准。

## 准备

- Windows 10/11 64 位；普通用户账号即可，安装不应要求管理员权限。
- 从 GitHub 对应版本的 Release 下载 `PaperPiggy-Setup-v<版本>.exe`。不要使用旧便携包：当前只发布安装器。
- 安装到本机可写目录，推荐默认位置或 `D:\PaperPiggy`。不要安装在 OneDrive、坚果云、Dropbox 等同步目录中。
- 准备一种文献来源：
  - Zotero 文库；或
  - 一个包含 PDF、EPUB、DOCX、Markdown、TXT 的测试文件夹。
- 若测本地检索，首启需要联网下载约 900MB 模型，解压后占用约 1.2GB；若测 SiliconFlow 云端检索，准备对应 API Key。

## 一、安装与启动

1. 运行安装器，确认不弹 UAC，可选择安装目录，并能创建开始菜单或桌面快捷方式。
2. 从快捷方式启动，确认直接出现 PaperPiggy 原生窗口：不能闪终端黑窗，不能只留下无窗口的后台进程。
3. 若系统缺少 WebView2，记录应用是否给出可理解的提示或正确回退到浏览器。
4. 确认安装目录内程序、数据、模型和 `0_Agent*` 工作区按“数据与程序同目录”设计落位；不要手工修改这些文件来让测试通过。

## 二、首启向导

1. **欢迎与环境**：文案完整、中文路径无乱码。
2. **选择检索方式**：
   - 本地模式：应明确提示“下载约 900MB，解压后约 1.2GB”；
   - SiliconFlow：应说明检索时联网、待检索文本会发送给服务商，并能测试 Key。
3. **连接文库**：
   - Zotero 模式应能从默认或自定义位置找到 `zotero.sqlite`；
   - 文件夹模式应能选择目录并识别五种受支持格式。
4. **准备检索与建立索引**：本地模式完成模型下载和校验后再建索引；云端模式不下载本地模型。失败时应显示服务端真实原因，而不是只报路径或 HTTP 状态。

## 三、核心功能

- [ ] 题录查找和全文检索均能返回结果，搜索词与结果不会因快速连续操作而错位。
- [ ] 浏览页的分类、筛选、排序、加载更多与详情正常。
- [ ] 对一篇可提取全文的文献执行深索，完成提取、嵌入和页级定位；扫描 PDF 的 OCR 在本机运行。
- [ ] PDF、EPUB、DOCX、Markdown、TXT 的状态和位置文案与实际格式一致。
- [ ] 综述库能打开来源、直接编辑正文、保存后标为已核验，并能改回未核验。
- [ ] Agent 页显示本机真实 MCP 接入命令，工具数来自后端；工作区文件夹可打开。
- [ ] 云端检索、对话和外部 AI 助手的联网边界说明与实际配置一致。
- [ ] 所有删除、清空、恢复和删除同步操作都有明确危险确认。

## 四、关闭与再次启动

1. 关闭窗口后检查 PaperPiggy 启动器和后端进程是否干净退出。
2. 再次启动，确认已有设置、索引、综述与 Agent 工作区仍在，且不会重新弹首启向导。
3. 从 Zotero 或来源文件夹新增一篇文献，执行“手动更新知识库”，确认只做增量更新并给出真实结果。

## 五、应用内更新

> 需要更新链路时，用一个较旧但已支持应用内更新的安装器开始，并只使用可丢弃的测试数据。首次获得更新功能的版本仍必须靠完整安装器安装。

1. 在设置页检查新版本，下载增量包并执行更新。
2. 确认更新过程不弹终端黑窗；应用关闭、替换代码并自动重启。
3. 确认数据、模型、索引、综述和 `0_Agent*` 均未被替换或丢失。
4. 若暂存版本无法导入或缺少新依赖，应放弃更新并保留原应用可用，提示改用完整安装器。
5. 若用户修改过安装目录下的 `.py`，确认旧代码被保留到应用提示的备份目录。

## 六、反馈材料

- 在应用的“设置 → 应用”导出错误日志。
- 记录安装器版本、Windows 版本、安装路径、文献来源模式、检索引擎，以及出错前的逐步操作。
- 截取应用内错误提示；若出现“点了没反应”，同时记录任务管理器中是否已有 PaperPiggy/Python 进程和可见窗口。
- 失败就按原样反馈，不要手工改安装目录或正式数据来掩盖问题。

# 数据格式与深索迁移 —— 保证以后开发的应用能复用已建的深索

> 目标：你已经花几小时建好的深索（19 万块向量）**永久可复用**。以后无论应用怎么迭代，
> 迁移都应是"拷数据"（几分钟），而不是"重跑嵌入"（几小时）。

## 一、深索数据长什么样（LanceDB 表 schema）
统一一张表 `chunks`，每行是一个"块"（`row_type` 区分题录/正文块）：

| 列 | 含义 |
|---|---|
| `chunk_id` | 块唯一 id（`<key>::<n>`） |
| `key` | Zotero item key（关联文献） |
| `vector` | **1024 维 float32 向量**（bge-m3，L2 归一） |
| `text` / `parent_text` | 命中段 / 所在父块原文 |
| `page` | 原文位置序号(int64)：PDF 为顺序页，其他格式为章节/段落单元 |
| `heading` | 所属标题或章节/行号定位 |
| `title/author/year/journal/doi/langid` | 题录字段 |
| `journal_tier` | 期刊分级 |
| `official_pages` | 官方页码 |
| `itemtype/has_pdf` | 类型/是否确有 PDF（兼容列；通用全文状态从 `papers.jsonl` 的 `has_fulltext/fulltext_format` 获取） |
| `row_type` | `meta`(题录) \| `chunk`(正文块) |
| `ingested_at` | 入库时间 |

配套（都在 data 目录）：`bm25/`(词法索引) `bm25_meta/`(题录词法) `meta/papers.jsonl`(题录全字段，含
`has_fulltext/fulltext_path/fulltext_format/fulltext_attachments`；主全文优先级 PDF→EPUB→DOCX→Markdown→TXT)
`state/embedded_keys.txt`(已入库 key) `categories/`(收藏夹+AI主题) `stats_cache.json` `index_manifest.json` `summaries/`(SAC摘要)。

## 二、★ 向量兼容红线（改了就作废，必须重跑）
已建向量能复用的前提——**嵌入管线一字不改**：
- 模型：**bge-m3**
- 后端：**ONNX-INT8**（`model_quantized.onnx`）
- 池化：**CLS**（取 `last_hidden_state[:,0]`）
- 归一：**L2 归一**
- 维度：**1024**，`max_length=512`
- SAC：摘要前缀**只拼进"嵌入文本"**，存表的 `text` 仍是原文（非对称设计）

> 只要这些不变，旧向量永远有效。**换 LLM、改 UI、改检索排序、加功能都安全**；
> 唯独动 embedder 的模型/池化/归一/维度会让全部旧向量失效。
> 见 [模型与职责.md](模型与职责.md)。⚠️ 本地 INT8 与 API 全精度是**两套向量空间**，不能混用同一张表。

## 三、已建深索现在在哪（都在原机器，不在本 OneDrive 项目里）
本 OneDrive 是**纯开发项目**，不含任何索引数据。你已建的深索在原机器：
- `D:\LocalKB\data\`　——LocalKB 的深索（**192,695 行**，由下面 RAG 库导入、复用向量）
- `D:\00Zotero知识库\rag\data\`　——最初的 RAG 知识库（原始 5 小时建的那份）

## 四、迁移怎么做（几分钟，不重跑）
未来"让应用用上已建深索"= 把索引数据放到应用的数据目录。安装器版的数据目录**就在安装目录内**
（`<安装目录>\data\`，靠 `portable.txt` 开关；比如 `D:\PaperPiggy\data\`）；
安装目录不可写时回退 `%LOCALAPPDATA%\PaperPiggy\`。两者都随更新不丢。

**方案 A：直接拷数据（最快）**
把 `D:\LocalKB\data\` 整个目录拷到目标 `<数据目录>\`（含 lancedb / bm25 / meta / state / categories）。
向量原样复用，秒级到分钟级完成。→ 到时我来执行（你说"进行迁移"时）。

**方案 B：从 RAG 库重新导入（换机器/重置时）**
跑 `import_fulltext.py`：靠 DOI/标题把 RAG 库的块映射到 Zotero item key、**复用向量免重嵌**（当初 99% 命中、约 1 分钟）。前提是那台机器上有 `D:\00Zotero知识库\rag\data`。

**跨机器**：索引数据（~1.2GB）不进 OneDrive。若要在新机器上也用这份深索，
把 `data\` 目录用移动硬盘/OneDrive 单独同步过去（和开发源码分开传）。或在新机器上重新建（选择性深索只做需要的文献）。

## 五、给未来会话/开发者的迁移清单
1. 确认目标应用的 embedder 仍是 bge-m3 ONNX-INT8 / CLS / L2 / 1024（红线未破）。
2. 确认后端一致（旧索引是 local 建的，就用 local；API 建的用 API）——看 `index_manifest.json` 的 `backend` 字段。
3. 拷 `data\` 或跑 `import_fulltext.py`。
4. 起 server，`/health` 应显示 `mode:full` + 正确篇数/块数即成功。

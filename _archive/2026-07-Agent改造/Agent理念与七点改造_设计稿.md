## 1. 一句话理念（贯穿全页的说明）

> **文献始终在你本机；agent 只是替你查库、写回综合与记忆；换任何 AI 助手都能接着用。**

三层文件，各司其职，都**人类可读**、都**留在知识库文件夹本地**：

- **你的原始文献**（PDF / Zotero）——只读，agent 不动。
- **`0_Agent交付物`**——agent 产出的成品（论文、资料汇编、周报）。
- **`0_Agent资料库`**——agent 干活要用的东西（**记忆**、技能、参考格式、交付模板、定时任务定义）。换 agent 时，新 agent 读这个文件夹就能无缝接上。

之上还有一层 **综述库（wiki）**：agent 把"你的库里就某个问题说了什么"沉淀成可复用的综合页——这层已经存在，本次只增强其"自动维护"（§4）。

**记忆 ≠ 综述库**，边界必须清楚（老库最大的病就是二者混淆）：
- **综述库（wiki）** = 文献综合层，回答"我的库里说了什么"。
- **项目记忆** = 项目决策/偏好/状态，回答"这个项目定了什么"。

### 2.1 放在哪、怎么不污染文献库（关键技术点）

- **folder 模式**（用户选了一个 PDF 文件夹）：`0_Agent*` 建在**受管文件夹内部**（贴合"在知识库文件夹里面"的心智）。**但** `folder_source.scan()` 是 `rglob("*.pdf")` 递归扫描，会把子目录里的 PDF 也吞进库。→ **必须在 `scan()` 加一条排除**：跳过名字以 `0_Agent` 开头的目录。否则交付物初稿、格式范本 PDF 会误入文献检索。（`folder_source.py:16`，唯一改动点，风险小。）
- **zotero 模式**（无单一受管文件夹）：落在应用数据目录同级 `%LOCALAPPDATA%\LocalKB\0_Agent交付物`、`...\0_Agent资料库`。
- 两种模式下，实际路径都由后端算好，前端只认"打开交付物 / 打开资料库"两个按钮，不暴露路径细节。

### 2.2 换 agent 也能读——靠什么？

**不是靠 skill（那是 Claude 专属格式），而是靠 MCP 接入时自动下发的一段通用指令。** 任何 MCP agent 一连上 LocalKB，`initialize` 就下发：

> "你的记忆在 `0_Agent资料库/记忆/项目记忆.md`，技能在 `技能/`，参考格式在 `参考格式/`，定时任务定义在 `定时任务/`。开工前先读它们；产出放 `0_Agent交付物/<主题>/`。"

现有代码已有这条通道（`mcp_server.py:72` 的 `instructions` 已在下发 wiki 规约），本次**扩写**它即可，跨 agent 零成本。

## 3. 逐点设计

### P1 · 交付物（`0_Agent交付物`）

- **默认交付形态**（承老库）：每主题一子文件夹，一条流水线 `检索原始材料 → 写作方案 → 初稿(md+docx) → 资料汇编 → 补全清单`。
- **每主题一个 `README.md`**：三段头（用途 / 引注规范 / 与其他材料的关系）——老库每份材料都有这三段，是"素材可回查"的关键。
- **交付形态模板**存在 `0_Agent资料库/交付模板/`，agent 照模板产出（用户可改模板 = 改交付形态）。
- **Agent 页**新增一张卡「📦 你的交付物」：显示最近 N 个主题 + 「打开交付物文件夹」按钮 + 一句提醒"**和你的 agent 商量清楚你要的交付形态**（篇幅/引注风格/要不要 docx）"。
- **优化点**（相对老库）：README 三段头模板化；引注纪律做成交付质量门（带页码、期刊分级优先、`【待核】`标记、每维度必有 gaps 小节）。

### P2 · wiki 自动维护（agent 会不会自己维护）

**现状**：wiki 从不自动生成；两个后台"探测器"（Lint 24h 体检、深索后 propose 建议）只写待办清单不动笔；agent 已被 MCP 指令要求当"维护者"，但真动手靠 agent 自觉。**设计上刻意拒绝全库自动预综合**（成本 2–3.5 亿 token），这条红线保留。

**要"更自动、优先级更高"，三招叠加（都不碰红线）：**
1. **把服务器已算好的"受影响页清单"暴露给 agent。** 现在 `wiki_suggestions.json` 只喂网页横幅、agent 拿不到——这是最大断点。新增一个 MCP 只读工具 `pending_wiki_updates`（读同一个 json），让 agent 深索完能直接拉到"这些页可能要更新"，不用自己重跑 propose。
2. **强化 `initialize` 指令**：把"维护 wiki"提为高优先级纪律（读完一篇必查 backlinks 判断是否标脏/重生；跑完定时任务必依新内容更新 wiki）。
3. **写进技能包和定时任务**：把"收尾更新 wiki"做成流程里的固定一步。

**信任护栏保留**：agent 写的标 🤖未核验、不覆盖你核验过的页、可一键删。

### P3 · "必须装 Claude Code 或 Codex" → 改成"任意 MCP agent"

MCP server 是通用 stdio JSON-RPC，**任何支持 MCP 的 agent 都能连**（Cursor / Cline / Windsurf / Gemini CLI / 通义灵码 …）。现在把 Claude/Codex 写死在 hero 标题、先决条件文案里，且把"通用 mcp.json"埋在第三个 tab。

**改法**：
- Hero 标题：`把文库接进 Claude Code / Codex` → **`把文库接进你的 AI 助手`**。
- 先决条件那行灰字：`需先装好 Claude Code 或 Codex` → **`你在用哪个 AI 助手，就接哪个；只要它支持 MCP 就行`**，配一句"没用过命令行 agent？**推荐 Claude Code**（最省心），装法见→"。
- 接入 tab 顺序改为：**推荐 Claude Code → 备选 Codex → 通用（任意 MCP 客户端）**，通用档不再叫"通用 mcp.json"而叫"其它 AI 助手"。

### P4 · "更多接入方式" 说人话 + 技能自动装

- **"🌐 自己写脚本"** = 不走 MCP、直接调本地 HTTP 接口（`127.0.0.1:8770/docs` 有全部接口+试调）。文案改成 **「不用 MCP？直接调接口」**，一句话说清"给不支持 MCP 的框架/脚本用"。
- **技能自动装（决策 4）**：
  - **Claude Code**：MCP server 被 Claude 拉起时工作目录就是用户项目根。`initialize` 时检测 `.claude/` 存在则把 `技能/localkb-paper`（源在 app 或 `0_Agent资料库/技能/`）**自动拷进 `./.claude/skills/`**；已存在且被用户改过（哈希不符出厂）就跳过不覆盖。
  - **其它 agent**：skill 格式不通用 → 把工作流要点并进 `initialize` 的 `instructions` 通用下发（现在只发 wiki 规约，扩写为"论文工作流 + wiki 维护"）。
  - **Agent 页**：技能那段从"复制路径"升级为 **「打开技能文件夹」按钮**（复用 `/setup/open_folder` 加目录参数）＋状态"已自动安装 ✓ / 未检测到，点这里手动装"。

### P5 · agent 专属资料库（`0_Agent资料库`）

- 内容见 §2 结构。核心是 **`记忆/项目记忆.md`**——把老库那份"宪法"产品化，但**吸取教训**：`项目记忆.md`(当前真相，短) 与 `变更日志.md`(历史) **物理分开**，前者机器可查、可自动刷新，别让它膨胀成流水账。
- **参考格式/**：承老库论文格式包（规范 + styles.xml + 范本 docx）。把"散文规范"升级为**可执行模板 + 校验器**（脚注 sz=18、eastAsia=宋体、脚注 vs 尾注两派、绝不重建 docx 以保 Zotero 引注域——这些坑固化成校验，别再靠人自觉）。★排除出文献扫描。
- **Agent 页**新增卡「📚 专属资料库」：说明"这是 agent 专属，放**参考格式、技能、记忆**等；agent 会自己维护、保持人类可读" + 「打开资料库文件夹」按钮 + 一句"换了 AI 助手也会自动读取这里"。
- `README.md` 由 agent 维护，列出资料库里有什么（人类可读索引）。

### P6 · 定时任务（应用登记+展示，agent 执行）

**分工（决策 2）**：应用**跑不了**联网研究（后端纯本地、无大模型、不联网）。所以——

- **任务定义** = `0_Agent资料库/定时任务/<任务>/任务.md`，人类可读、agent 中立。字段建议：
  ```yaml
  ---
  名称: 少年司法周报
  频率: 每周一 08:30
  启用: true
  ---
  搜什么：过去 7 天内中国少年司法领域的新动态（立法/司法解释与典型案例/学术新论文/其他）
  输出：写 0_Agent交付物/定时任务/少年司法周报/周报_<日期>.md；append 重点摘录.md（最新在上）
  收尾：把周报并入知识库；依新内容更新相关 wiki 页；淡季如实标"本周无新增"，不硬凑
  ```
- **执行** = 你的 agent（Claude Code 等）读这个 `任务.md`，在**它自己的日程系统**里建 cron 跑。**换 agent 时**，新 agent 读同一个 `定时任务/` 目录、在自己的系统里重建——这就是"换 agent 也能衔接"。
- **成果** = `0_Agent交付物/定时任务/<任务>/`（周报 + 重点摘录台账），承老库形态。
- **入库 + 更新 wiki**：任务收尾时 agent 调 LocalKB 的 MCP/HTTP 把周报入库，并依时效内容更新 wiki（P2 第 3 招）。时效性强的周报正是 wiki 的活水。
- **Agent 页**新增卡「⏰ 定时任务」：读 `0_Agent资料库/定时任务/` 显示"当前有哪些任务、多久一次、成果去 `0_Agent交付物/定时任务/`、会更新 wiki"。
- **迁移提示**：你现在那条 `weekly-juvenile-justice-scan` 入的是**老库 rag**；将来给它一份指向 LocalKB 的 `任务.md` 模板即可切过来。

### P7 · 自动更新（按天 + 补跑 + 删除同步）

**现状**：进程内线程、分钟级(15/30/60/180)、只在应用开着时跑、只加不删、不补跑。

**改造（决策 3）**：
- **调度语义**：`settings.json` 的 `auto_update` 从 `{enabled, interval_min}` 扩为
  `{enabled, interval_days: 1–30, at_time: "07:00", catch_up_on_launch: true, delete_sync: false}`。
  循环里由"上次成功时间 + interval_days 天、到 at_time"算下次触发（需引入按天/按时判断，无需完整 cron 库）。默认 **每天 7:00**。
- **补跑错过的**：启动时若"上次成功更新距今 ≥ interval_days"，在启动后（避开冷启）补跑一次——等价老库靠 Windows 计划任务实现的"关机也补"，本应用做不到关机跑，但能做"开机补"。
- **删除同步**：源里删掉的文献自动清出库。folder 模式已有雏形（`folder_ingest` 的 live 集合差集 + `_purge_db_rows`），扩成正式 purge；zotero 模式历史是 add-only，删除检测较难，作为**可选开关**默认关、UI 加提醒。
- **UI**：设置里把 15/30/60/180 下拉换成 **天数(1–30) + 时间选择器 + 两个勾选（补跑默认开 / 删除同步默认关+警示）**。
- **冲突**：构建有全局锁 `_BUILD_LOCK`，重建那几秒检索短暂返回"稍候再搜"；按天 + 早 7 点默认已基本避开用库时段，风险低。（残留竞态 B2 已知，非本次范围。）

## 4. 老库还能引入什么（只搬模式，不搬私人内容）

| 类型 | 老库来源 | 引入为 |
|---|---|---|
| 项目/交付流水线 | 交付物/ | §P1 `0_Agent交付物` + 五段式检索 schema + 三段头 README |
| 监测/情报型 | 少年司法周报/ | §P6 定时任务 + 周报/重点摘录累积台账 |
| 模板/样式包 | 论文格式/ | §P5 `参考格式/`，升级为可执行模板+校验器 |
| 项目记忆"宪法" | CLAUDE.md | §P5 `记忆/项目记忆.md`（当前真相与变更日志分离） |
| 引用纪律 | CLAUDE.md 检索约定 | 交付质量门：带页码、期刊分级优先、`【待核】`、gaps 小节 |

**分发前必须参数化的私有耦合**（老库教训）：作者/基金块（林树建/24CFX098 之类）、绝对路径 `D:\`、无 API key 导致的 SAC 漂移、WMI/Windows 计划任务等机器耦合——分发版都要抽象掉。

## 5. 理念上的 3 点提醒（老库踩过的坑）

1. **记忆 vs wiki 分清**（§1）。老库 CLAUDE.md 膨胀到 23KB、从"宪法"退化成"变更日志"，新会话要读半天才知现状——**当前真相与历史流水账物理分开**。
2. **"自动+高优先级"配住护栏**：默认采纳但标未核验、不覆盖已核验、可一键删。别让 agent 自主写变污染。
3. **可移植性靠 MCP，不靠 skill**：skill 只是 Claude 的便利层；跨 agent 的共同底座是"MCP server + 人类可读的资料库文件夹"。

## 一、wiki 三环扳机（gist 对标补缺——让 wiki 转起来）

| 项 | 内容 |
|---|---|
| **lint 定时器** | 自动更新线程每 24h 顺带跑一次 `W.lint()`（纯读零成本，**不随自动更新开关关闭**），结果落 `data/state/wiki_lint.json`；首页库总览出现「🩺 综述库有 N 处待理顺 →」角标，点击直达体检面板。gist 点名的头号失败模式 drift 从"靠人记得"变成"系统盯着" |
| **Ingest 扳机** | 深索成功后后台自动算"这批新文献影响了哪些综述页"（队列批次与整库深索两条路径都挂了钩，单批上限 20 篇），综述库页顶出现「📬 N 篇新深索文献可能影响你的综述页」横幅：文献题 + 受影响页 chips + 忽略 |
| **时间线** | 综述库工具栏「🕘 时间线」：解析 wiki 的 git 提交历史（无 git 退快照目录），每条=时间 + 🤖/👤 + 动作 + 页链接——gist 的 log.md 等价物。顺带修正了 update_page/set_links 的历史标注（此前 agent 改动会被标成人） |
| **来源门槛** | 综合页落盘 provenance 前按"相对最高分"过滤离题命中（本地/API 两种分数尺度自适应，至少保留 2 条）——真实案例：digest 页曾混入测试文件和离题文献。ctx 与来源列表用同一份过滤名单，编号不再错位 |
| **正文互链** | 正文支持 `[[page-id]]` / `[[page-id|文字]]` 渲染成页内跳转链接（行内代码里的原样展示、XSS 安全）；lint 新增第七类检查：正文断链 |
| **schema v2** | WIKI.md 升级：新增 gist 点名的「新建页 vs 原地编辑」判定规则 + 正文互链约定。沿用既有哈希升级机制：你没手改过就自动升级并留档 WIKI.v1.md，手改过则不动 |

## 二、法学检索增强

| 项 | 内容 |
|---|---|
| **法条按"条"切分** | statute 文献先按行首「第X条」切条界（句中交叉引用不误切、"第三条例外"有负向前瞻防护），条内再句聚合；每块 heading=条号，检索命中直接可引 |
| **法规/报告引注** | cite_format 新增两式：statute→「《法名》（YYYY年）第X条」、report→「机构：《报告名》（YYYY年），第X页」；不认识的类型回退期刊式 |
| **同义词查询扩展** | 出厂 77 组法学同义组（少年司法↔未成年人司法、辩诉交易↔诉辩交易…宁缺毋滥，已剔除"合规不起诉≠附条件不起诉"、"dpa 撞数据保护机构"这类似同实异组）；只扩 bm25 查询词袋，零重建成本，`SYN_EXPAND` 可一键关；你可在 `data/legal_synonyms.txt` 自行加组（每行一组、顿号分隔） |
| **词典治理** | jieba 法律词典：来源条目按 itemtype 白名单（网页/博客的 keywords 不再进词典——"EXCEL技巧"们出局）、丢书名号/教程词、keywords 长度上限 8；叠加出厂 207 个法学核心术语 |
| **法条时效** | statute 标题/extra 里的「已废止/已失效」「YYYY修正/修订」自动识别：检索卡和浏览卡显示红/橙徽标提醒核对现行版本；**已废止**在三种排序下统一降权 ×0.5（沿用负分安全写法） |

## 三、Agent 接入（蓝图 G1/G2/G4 全部落地）

**MCP 工具 21 → 28 个**（19 读 / 9 写）：

- **`verify_claim`（G1 核验器，旗舰）**：一句论断 → supported / mismatch / not_in_lib 三态 + 证据（题名/页码/支撑句/分数）。判定刻意保守：mismatch 只留给"引文定位到了原文但与声称不符"的硬证据，其余不确定一律 not_in_lib 并恒附「库内未覆盖 ≠ 论断为假」（铁律三）
- **`format_citation`（G2 出口）**：拿 key+页码 → 规范引注 + missing_fields；引领词（参见/见）由作者自定
- **`locate_quote`**：核对某句话是否真在原文、在第几页（归一化匹配，支持模糊）
- **`add_source`**：agent 可把本地 PDF 路径入库（folder 模式；查重、只加不删、题录待人工核对）——入库闭环补上
- **`get_source_meta`** / **`similar_sources`** / **`whats_new`**：单篇全档、相似文献、增量感知
- list 系工具全部支持分页；7 个工具附带 structuredContent 结构化返回（MCP 2025-06 规范）；`localkb://page/{id}` resource template 补声明
- **G4 披露**：`POST /research/disclosure` 按页面 frontmatter 生成《生成式 AI 使用声明》
- **人用入口**：综述页详情弹窗底部新增「🔎 核验一句话」小工具（不经 agent 也能用核验器）

**skill 包**：`skills/localkb-paper/SKILL.md`——法学论文写作工作流（意图澄清→迭代检索→提纲人工闸→分节起草绑源→verify_claim 逐条核验→排脚注→wiki 沉淀→披露提醒）。安装：把 `LocalKB/app/skills/localkb-paper` 复制到项目 `.claude/skills/` 下。Agent 页新增「更多接入方式」卡（含 OpenAPI 文档地址 `http://127.0.0.1:8770/docs`、skill 安装、路径复制按钮）。

## 六、生效条件与已知限制

- **法条按条切分只对新切块的文献生效**：已有 chunks 的文献断点续跑会跳过；需要时删掉 `data/chunks/` 对应文件重跑深索。
- 同义词扩展的最佳效果在词典重建后（第 2 步的更新会顺带完成）；重建前走 substring 兜底通道，效果稍弱但可用。

## 动手前必须知道的地基事实（排查已确认）

1. **「更新知识库」按钮现在是坏的**：前端 POST `/build`，但 `server.py` 无此路由，404 被静默吞掉。Phase 0 要先补 `/build`→`_run_build("all")` + `/build/status`。
2. **「检索结果不可复制」根因是 `launcher.py:35` 缺 `text_select=True`**（pywebview 默认整窗禁选），不是 CSS。一行修，重启验证。
3. **期刊分级有两套系统**：用户看到的主徽标来自旧扁平表 `journal_tiers.json`（无学科、个人决策全局烤死）；设置里的「学科」只驱动新引擎 `journal_grading` 的次要权重徽标。F38-B 治本 = 主徽标改吃新引擎、**无需重建索引**（详见该设计文档）。
4. **新用户进浏览满屏是 0**：收藏夹树计数用的是「已深索数」，深索是后置动作 → 首次全 0，像空库。Phase 0 改成「已深索/总数」双数字。
5. **两套代码树**：`LocalKB源码`（开发源）→ `sync_app.ps1` → `LocalKB\app`（打包副本）。只改源码、跑同步。
6. **文件夹模式的固有代价**：非 Zotero 用户的题录靠 LLM 从 PDF 正文抽（每篇 1 次 LLM，200 篇约 5–15 分钟），且 `journal` 字段多不可靠（全部标 `needs_review`）——这是设计已知取舍，照文档做兜底，别试图秒级。

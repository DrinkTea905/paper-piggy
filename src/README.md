# PaperPiggy · 本地文献知识库

把你的文献（Zotero 库，或一个装满全文文件的文件夹）变成一个**本地运行的知识库应用**。支持 PDF、EPUB、DOCX、Markdown、TXT，不支持 HTML：
语义 + 关键词混合检索、按来源评价加权、**原文位置可回溯引用**，主要数据保存在你自己的电脑上。

- 想要 **AI 问答/写综述**：填一个你自己的大模型 API Key，基于本地文献问答，答案带原文定位引用。
- 想要 **给 Claude Code / Codex 等 Agent 当检索工具**：内置 MCP 接口，Agent 能直接查你的文献库（见「进阶：接入 Agent」）。

---

## 启动

**双击 `启动.bat`** → 稍候片刻，弹出 PaperPiggy 的**原生应用窗口**（不是浏览器标签）。

- 应用先读取轻量的知识库目录，检索模型和倒排索引在**第一次搜索时**才加载；本地模型通常要多等几秒，API 模式更快。默认连续 10 分钟没检索会自动释放，可在「设置 → 检索 → 检索内存」调整。
- 若你的电脑缺少 WebView2 运行时，程序会**自动改用系统浏览器**打开，并弹提示说明；此时**关闭浏览器标签不会退出后台服务**。
- 启动失败时会弹窗提示，并把详情写进 `logs/server.log`（把它发给技术支持即可定位问题）。

---

## 第一次使用：跟随向导

第一次打开会有一个**设置向导**，一步步带你把库建起来。两条路，二选一：

1. **我用 Zotero** → 选中你的 Zotero 数据目录，PaperPiggy 直接读 `zotero.sqlite`（不需要装任何 Zotero 插件），把题录和受支持全文附件接进来。同一题录有多个附件时，主全文按 PDF → EPUB → DOCX → Markdown → TXT 选择。
2. **我只有一批全文文件** → 选一个文件夹，PaperPiggy 把其中的 PDF、EPUB、DOCX、Markdown、TXT 导入建库。

> 题录 = 题名·作者·年份·期刊等书目信息。

向导还会让你二选一挑**检索引擎**（下一节）。跟着走完即可进入知识库。

---

## 两种检索引擎，怎么选？

向导第 3 步让你选一种，两者中性并列、按需要挑：

| | **本地引擎** | **API 引擎** |
|---|---|---|
| 是什么 | 在你电脑上跑的开源模型（bge-m3 + reranker） | 调用云端向量服务（如硅基流动） |
| 要不要联网 | 建好库后**完全离线** | 每次检索都要联网 |
| 前期成本 | 首次要下载约 **1.2GB** 模型 | 免下载，但需要一个 **API Key** |
| 适合谁 | 追求离线/隐私、网络能下大文件 | 不想下大文件、机器较弱 |

- 选**本地引擎**：向导会下载模型。下载支持断点续传；若网络实在下不动，可随时**改用 API 引擎**立即开始用。
- 选 **API 引擎**：在设置里填一个向量服务的 Key 即可。

> 🔒 你填的任何 API Key 只存在本机（浏览器 localStorage + 本地服务内存），除了你选的服务商本身，不上传任何第三方。

## 日常使用

- **📚 文献**：既可按题名/作者查题录，也可输入研究问题做混合检索；结果带来源评价、页码/章节/段落/行号和原文命中段。
- **💬 对话**：填好大模型 Key 后提问，答案**只依据本地文献**，每个论点带 `[编号]` 引用，可展开来源原文；综述可**一键导出 Word**。
- **📚 浏览**：按收藏夹 / AI 主题 / 你自建的分类浏览文献；可把文献「加入分类」以便聚焦检索或喂给 Agent。
- **⟳ 更新知识库**：在 Zotero / 文件夹里加了新文献后，点它增量更新（只处理新增，已入库的跳过）。

### 三档索引（了解即可，向导会自动安排）

1. **词法档**：秒级就绪，先能按关键词搜。
2. **语义档**：分钟级，加入向量语义检索。
3. **全文深索**：把全文附件拆成小段并向量化。PDF 定位到具体页码，EPUB 定位到章节，DOCX / Markdown 定位到标题段落，TXT 定位到行号。深索较慢，可对选中文献按需触发。

> 扫描件 PDF（图片、无文字层）会在深索时自动做**本地 OCR**：只识别没有文字的页面，
> 结果仅写入 PaperPiggy 索引，不修改原 PDF、不上传文献。附件不存在或 PDF 损坏时会单独提示，
> 不再误报为「需 OCR」。

## 常见问题（FAQ）

**Q：双击 `启动.bat` 没反应？**
A：多半是没找到 Python 解释器。`启动.bat` 会在找不到时弹一个黑窗提示并暂停；按提示确认本目录下有 `python\` 子目录或 `.venv\`，或系统已装 Python。

**Q：等了很久没出现窗口？**
A：窗口本身不再等待检索模型加载。若始终不出现，看 `logs/server.log` 末尾的报错；常见原因是 WebView2、端口或数据目录权限异常。模型在第一次检索时才加载，本地模式届时多等几秒属于正常现象。

**Q：提示端口被占用？**
A：说明 `8770` 端口被别的程序占了。关掉占用它的程序后再启动。

**Q：对话/综述报「密钥无效或余额不足」？**
A：检查你填的大模型 API Key 是否正确、账户是否有余额。

**Q：数据存在哪？会不会污染我的 Zotero？**
A：所有索引/数据都写在应用自己的 `data` 目录，**绝不写入你的 Zotero 库**。Windows 安装版默认与程序同目录（例如 `D:\PaperPiggy\data`）；若安装位置不可写才回退 `%LOCALAPPDATA%\PaperPiggy\data`。删掉 PaperPiggy 的 `data` 不影响 Zotero 原文献，但会丢失索引、综述和设置，操作前请先备份。

## 进阶：接入 Agent（Claude Code / Codex）

PaperPiggy 内置 MCP 服务，能让 Claude Code / Codex 直接检索你的文献库、把综合结论回填复用。

**最省事的接法**：打开应用 →「🤖 Agent」页，那里有**本机真实路径已填好**的一键接入命令，直接复制即可。
原理与工具清单见 `MCP接入说明.md`（文档里的路径是占位示例，请以 Agent 页的真实命令为准）。

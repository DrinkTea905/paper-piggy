    1

    主线 · PaperPiggy + Agent

    PaperPiggy 是 Agent 的文献知识引擎。它管理题录、全文、原文位置、检索摘要和综述库；Agent 负责检查知识库、检索文献、精读原文、核验论断、整理成果和持续维护。

    你负责说明研究问题、范围、交付物和证据标准。Agent 负责选择工具、安排步骤，并把可复用成果保存到本机的交付物、综述库和项目记忆中。

    2

    完成基础建库

    首次向导依次完成：
    · 选择本地检索或 SiliconFlow 云端检索；
    · 连接 Zotero，或选择一个装有 PDF、EPUB、DOCX、Markdown、TXT 的文件夹；
    · 建立题录和语义索引。
    第一版索引可用后即可连接 Agent，全文深索可以按课题分批完成。

    3

    连接 Agent

    打开顶部「🤖 Agent」页，选择 Claude Code、Codex 或通用 MCP 客户端，复制应用生成的本机接入命令，在 Agent 会话中运行。接入后先让 Agent 查看知识库状态并搜索一个库内主题。

    4

    布置第一项研究任务

    把四件事说清楚：研究问题、资料范围、交付物、证据标准。例如：
    “研究自动化行政中的说明理由义务。先检查库内覆盖情况，再按理论基础、制度方案和反对意见分组；精读代表文献，关键判断给出原文位置，最后生成三级提纲并列出资料缺口。”

    5

    让 Agent 安排深索与检索摘要

    深索把全文拆成可检索片段，并保留页码、章节、标题段落或行号。扫描版 PDF 会在本机完成 OCR。检索摘要帮助系统识别整篇文献的主题，不是原文摘要，也不替代深索。

    让 Agent 围绕当前课题挑选最相关的一批文献，分批完成深索和检索摘要。大型文库可在电脑空闲时逐步扩展。

    6

    让成果进入交付物和综述库

    论文初稿、资料汇编、报告等成品保存在交付物；研究方法、项目记忆、参考格式和定时任务保存在资料库；需要长期复用和更新的结论进入综述库。综述页保留来源、核验状态和版本历史。

    7

    让 Agent 维护知识库

    在 Zotero 加入一批文献后，可以直接说：
    “维护我的 PaperPiggy 知识库。检查新增题录、全文附件、深索、OCR、检索摘要、待更新综述、孤儿页、断链和过时内容；安全事项直接处理，需要研究判断的事项集中说明。”

      5

      综述库（wiki）：把问答沉淀成越用越厚的知识

      AI 每答一个问题、读完一篇文献，可把结论写成一页带引用的综述，下次直接命中、不重复劳动。综述页会按来源与现有 AI 主题的重合度自动归类；你也能新建主题、手动固定、搜索和排序。它还会主动维护：新文献可能推翻旧结论时会标记「过时」并写明理由。AI 写的页都标 🤖 未核验、不覆盖你亲自核验过的页；未核验页在检索里会被降权，不会盖过原始文献。打开一页后可沿目录阅读、核验、查看来源与历史；删除自定义主题只会恢复自动归类，不会删除综述。

      6

      深索：让回答回溯到原文位置

      已深索的文献 AI 能读到带原文位置的正文；未深索的只有题录 + 摘要。想某方向问答更扎实，先去「📚 文献」深索它（拆成可检索的小段，可后台慢慢跑，不影响使用）。PDF 没有文字层的页面会自动做本地 OCR；不上传文献、不修改原文件。附件丢失或无法读取会分开提示；文献页筛选器可按正文来源和深索状态查看，每项直接显示当前范围的篇数。

      💡 可选增强：检索摘要——深索完，可让 AI 给每篇写一段检索摘要（⚙ 设置 → 建库 → 检索摘要），检索更容易命中这篇；不生成也不影响读取和引用已深索全文。摘要必须通过质量检查才算完成；文献页可筛选“摘要有效 / 异常 / 缺失”。选择“交给 Agent 生成”后，让 Agent 维护知识库时，它会读取正文、修复可处理的缺失/异常摘要并重嵌入；不会改走 PaperPiggy 自动生成摘要的 API。

      🔎 Agent 做发现型检索时会优先铺开不同文献，每篇最多带回 2 段，避免一篇文献挤满结果；选定关键来源后，再做定向深读 / 论断核验，此时可以读取该篇更多证据。内置“写论文与综述”“跨学科发散与补文献”工作流已经按这个顺序执行。

      🧠 检索内存

      PaperPiggy 平时只保留轻量的库目录；第一次检索时才加载检索模型和倒排索引。 连续一段时间没有检索后，会自动释放它们，给其他软件腾内存。默认 10 分钟，正在进行的 界面检索、对话检索或 Agent 检索绝不会被中途释放。

      释放只影响临时内存，不会删除文献、索引或设置；下次检索会自动重新加载。

      怎么选：内存紧张建议 5–10 分钟；经常连续检索建议 30–60 分钟；选“不自动释放”会减少等待， 但本地模型和索引会持续占用更多内存。释放后第一次检索需要重新准备，本地模型通常比云端检索等得更久。

      ⚖️ 来源评价学科

      决定如何把期刊目录与各类文献性质映射为四档来源评价。整库统一一个学科，改了下次检索即生效、不用重建。

      ① 深索提取全文并拆成可检索小段，让文献能精读和定位原文；② 检索摘要再给整篇文献配一段 ~150 字 AI 摘要，让系统更容易先找到这篇。没有摘要不影响读取已经深索的全文；它也不是题录里的原文摘要，不能替代深索。这里选择由谁生成：

      🔄 自动更新知识库

      默认每天早 7:00 增量更新；关掉就只能用顶栏「⟳ 手动更新知识库」。

      Zotero 条目至少要有 PDF、EPUB、DOCX、Markdown 或 TXT；没有可读取全文附件的纯题录不进库。多附件按 PDF → EPUB → DOCX → Markdown → TXT 选主全文。改后点顶栏「手动更新知识库」生效。

      🎨 外观（主题 / 字号）

      左上角太阳/月亮可快速切换明亮与夜间模式；这里也可选择跟随系统。字号偏小可调大，改动即时生效、存本机。

      💾 备份与恢复

      把丢了就再也没有的东西打成一个 zip：综述库、Agent 写的论文与记忆、收藏夹、 文献元数据、来源评价数据（其中部分结果花过 API 钱）。
      想要云备份？把下面的备份位置指到 OneDrive / 坚果云里的一个文件夹就行—— 云盘同步一个静态 zip 是安全的。⛔ 但千万别把整个数据目录丢进云盘： 向量索引是持续读写的数据库，云盘同步它会锁文件、造重复副本，早晚把索引搞坏。

      🧹 清空并从头重建索引

      想把整个索引推倒重来时用（比如库里有重复文献要合并、或深索乱了想重跑）。 它只清索引：全文提取、切块、向量、bm25、深索进度标记、整库状态。
      ✅ 明确保留：综述 wiki、收藏夹分类、来源评价数据、检索摘要（花过 API 钱的）、 页码映射、你的设置、0_Agent 工作区。
      清空后按提示点顶栏「⟳ 更新知识库」重建题录+语义（会自动去重、按“只导入有全文附件”重新导入）， 再到首页点「深索全部」重新深索。

      ⚠ 向量索引会被丢弃、需要重新嵌入（1400 篇要几十分钟到几小时、 云端检索还要重新调用一次嵌入接口）。强烈建议先在上面「💾 立即备份」打一份（勾上“包含向量索引”）， 万一想反悔能整包恢复。旧的提取/切块会移到一个备份目录、可手动捞回，但向量表是直接丢弃的。

      确认要清空索引并从头重建吗？综述、分类、分级、设置、Agent 工作区都会保留， 但向量索引会丢弃、需重新嵌入。这一步不可逆（除手动从备份目录捞回提取/切块）。

      ⬆️ 应用更新

      通常应用内升级只替换程序本体并自动重启。若提示“需完整安装器”，请下载后直接 覆盖原安装位置，不用卸载。两种方式都不动索引、综述、设置和 Agent 文件。

      国内下载慢/超时？填个镜像地址兜底

      检查更新走 GitHub API（小、通常能连）；只有下载增量包（约 1MB）会用镜像兜底， 完整安装器仍从官方 GitHub Release 下载。 把 paper-piggy-app-<版本>.zip 和同名 .sha256 传到 Gitee Release 或对象存储，把它们所在的目录前缀填这里（以 / 结尾）。留空=只走 GitHub。

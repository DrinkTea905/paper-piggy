把下面每个目录整理成一个 JSON 文件，放到 `journal_grading/catalogs/<id>.json`（种子/随源码分发），
用户机上可编辑的副本放 `<DATA>/journals/<id>.json`（优先，随索引走、app 更新不覆盖）。
`<DATA>` 与 config.py 同规则：环境变量 `LOCALKB_DATA`，否则 `src/data`。

## 统一文件格式（每个目录一份）

```json
{
  "_meta": { "catalog": "cssci", "version": "2023-2024", "source": "...", "updated": "2026-07" },
  "journals": [
    { "name": "中国法学", "issn": "1003-1707", "level": "来源" }
  ]
}
```

- `name`：刊名（中文用规范全名；外文用官方刊名）。匹配时会自动归一（去《》空格、全半角、简繁、拉丁小写）。
- `issn`：主 ISSN，`xxxx-xxxx`（可留空，则只按刊名匹配；**强烈建议尽量补全**，ISSN 命中最稳）。可另加 `eissn`。
- `level`：**必须**取该目录在 `grading_config.json > catalogs[id].levels` 里的枚举值（见下表「level 取值」列），否则该条不会被 map 接住。
- `_meta.version`：会写进每条命中的解释链（explain.hitCatalogs[].version），便于追溯用了哪版名单。

## 要点

- **SJR 是外文主力**：`ssci` 的 Q 分区不免费；没有 JCR 权限就把外文分区都放进 `sjr.json`（Q1–Q4），
  方案里 `ssci.Q*` 缺位会自动回退到 `sjr.Q*`，不报错、不影响主档位。
- **只需先补你选定学科用得到的目录**：本库锁 `law`（法学）→ 优先 `clsci`、`cssci`、`pku`、`warning` 四个即可开跑；
  外文可选补 `ssci`/`sjr`。其余目录（abs/ft50/utd24/fms/ahci/erih/ami）留空文件也不报错，等扩到相应学科再补。
- **认可顶刊（T1）已内置**：10 学科的 T1 顶刊（带 ISSN）已在 `grading_config.json > recognizedTopLists`，
  无需下载即可让顶刊定档 T1；但仍需该刊出现在对应名录里（如 `cssci.来源`）才能触发 `@recognizedTop`。
- **ISSN 存疑项**：规格标注「现代外语(1003-6105)、外国文学研究(1003-7519)」为通行记载值，
  入库前建议再核刊号页；实际匹配以你放进目录文件里的权威 ISSN 为准。

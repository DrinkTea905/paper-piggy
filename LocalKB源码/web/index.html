    ③ 内置工作流：带页级引注的资料汇编

    PaperPiggy 给 AI 装了一套「半自动研究助手」工作流，专门帮你把库里可用的材料按选题梳理成 带期刊印刷页码引注的资料汇编：

    1. 选题拆解 + 大纲：给它一个研究方向，它检索你的库、拆出关键词与核心命题、生成挂着 ★核心/☆辅助 文献的三级大纲。
    2. 分节综述（引注到印刷页）：对每个子题召回证据、合成综述，每个论点后紧跟「作者《题名》，期刊，第 X 页」——X 是读者翻期刊看到的那一页，不是 PDF 顺序页。
    3. 诚实标缺口 + 建议补文献：每节末尾如实标「资料缺口」，并从脚注里挖出你库里缺、值得补的文献（按期刊层级排）。

    直接对它说：「用 build_digest 帮我就『认罪认罚从宽对司法信任的影响』做一份带页码引用的资料汇编」 或 「先 research_outline 给我一个大纲」。产出会存进「📖 综述库」，随时复用。

    （完整的「一键写整篇论文」工作流在后续版本；当前先把最扎实的「材料梳理 + 页级引注」做透。）

    ④ 关于「深索」——AI 能读到多深，取决于这个

    AI 检索时，已深索的文献能返回精确到页码的正文段落；未深索的只能返回题录+摘要。 想让某个方向的问答更扎实，先去「📚 浏览」把该收藏夹/主题里有 PDF 的文献深索了。

    💬 自带对话：基于本地文库、带页码引用

    它直接读你的库回答、每句可回溯到来源页码，随手问答很顺手；一次会看少量来源，答案默认不自动沉淀（可手动「保存此答案」）。

    想更进一步——跨会话累积、系统化综述、把结论沉淀复用？可以再接「🤖 Agent」：让 Claude Code / Codex 也调你的库。两条路都可用，按需选。

    用对话：在下面「模型设置」里填一个 LLM 的 API Key（DeepSeek / 硅基流动 / Kimi / 智谱 / OpenAI 皆可）即可开问。硅基流动可复用检索引擎已配好的 key。

    ⚙ 模型设置（对话用；检索不需要 key）

    🔒 Key 只存本机浏览器（localStorage）与本地服务内存，不上传第三方（除你选的模型服务商）。

    💡 硅基流动有免费模型：对话推荐 Qwen/Qwen3-8B。选「硅基流动」且此处留空时，自动复用「设置→检索引擎」里的 SiliconFlow key。

    ⚖️ 期刊分级学科（整库锁定一个学科）

    决定用哪套学术标准，给检索结果里的期刊打「来源门槛」（0–1 分，越高越权威）。整库统一用一个学科。改了下次检索即生效，不用重建。

    「法学（开发者增强）」是开发者的私人法学期刊分级（含台湾刊 / 顶尖外文法评等），欢迎试用。想调具体分数或顶刊名单属于高级微调，告诉开发者即可。

    🧬 深索摘要（自动生成 AI 摘要提升检索）

    默认复用你在「检索引擎」里配的 API key（SiliconFlow 免费即可），无需额外配置。开关一点即存。

    🤖 用 Agent 驱动深索时也一样：只要这里开着，无论你在应用里深索、还是让 Claude Code / Codex 触发深索，服务端都会自动补摘要——不需要 agent 额外做什么。

    高级：单独指定摘要用的服务商 / key / 模型（可选）

    🔎 检索引擎（嵌入 / 重排）

    检索用的向量引擎，可随时切换或改 Key。⚠️ 已建过库后更换引擎（本地↔API、或换模型），旧向量不兼容，需点「更新知识库」重建。

    👉 还没有 Key？去 SiliconFlow（硅基流动） 登录即领免费额度。填这一个 Key，检索、对话、深索摘要、文件夹抽题录都能用它的免费模型；够用。想更进一步可再接 Agent。

    高级：嵌入 / 重排模型（一般不用改）

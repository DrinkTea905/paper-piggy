      ③ 内置工作流：带页级引注的资料汇编

      给 AI 装了套「半自动研究助手」，把库里材料按选题梳理成带期刊印刷页码引注的汇编：

      1. 选题拆解＋大纲：检索你的库，拆关键词与核心命题，生成挂 ★核心/☆辅助 文献的三级大纲。
      2. 分节综述（引注到印刷页）：每个论点后紧跟「作者《题名》，期刊，第 X 页」——X 是期刊印刷页，非 PDF 顺序页。
      3. 诚实标缺口：每节末如实标「资料缺口」，并建议库里缺、值得补的文献。

      直接说：「就『…』帮我做一份带页码引用的资料汇编」 或 「先给我一个大纲」。产出存进「📖 综述库」随时复用。

      ④ 关于「深索」——AI 能读到多深，取决于这个

      AI 检索时，已深索的文献能返回精确到页码的正文段落；未深索的只能返回题录+摘要。 想让某个方向的问答更扎实，先去「📚 浏览」把该收藏夹/主题里有 PDF 的文献深索了。

      ⑤ 它默认怎么动你的库？

      默认是「只读检索 + 另存综合」：查库、读页码都是只读的，不会改动你的原始文献；它写回的综述会新建一页并标 🤖「未核验」，默认可被检索命中、但排序不会盖过原始文献，不会覆盖或修改你已有的内容，不满意随时在「📖 综述库」一键删。深索、更新这类会动索引的操作，也只在你（或你明确让它）触发时才做。 AI 写回时会遵守本机的写回规约（引用格式 / 来源纪律），规约文件在：

      💬 自带对话：基于本地文库、带页码引用

      它直接读你的库回答、每句可回溯到来源页码，随手问答很顺手；一次会看少量来源，答案默认不自动沉淀（可手动「保存此答案」）。

      想更进一步——跨会话累积、系统化综述、把结论沉淀复用？可以再接「🤖 Agent」：让 Claude Code / Codex 也调你的库。两条路都可用，按需选。

      用对话：在下面「模型设置」里填一个 LLM 的 API Key（DeepSeek / 硅基流动 / Kimi / 智谱 / OpenAI 皆可）即可开问。硅基流动可复用检索引擎已配好的 key。

      ⚙ 模型设置（对话用；检索不需要 key）

      🔒 Key 只存本机浏览器（localStorage）与本地服务内存，不上传第三方（除你选的模型服务商）。

      💡 硅基流动有免费模型：对话推荐 Qwen/Qwen3-8B。若你在「⚙ 设置 → 检索引擎」里填过 SiliconFlow Key，这里留空即可直接用，无需重填。

      设置

      改动即时生效、无需「保存」。切换检索引擎（本地↔API）会让旧向量不兼容，改动前会二次确认。

      ⚖️ 期刊分级学科（整库锁定一个学科）

      决定用哪套学术标准，给检索结果里的期刊打「来源门槛」（0–1 分，越高越权威）。整库统一用一个学科。改了下次检索即生效，不用重建。

      「法学（开发者增强）」是开发者的私人法学期刊分级（含台湾刊 / 顶尖外文法评等），欢迎试用。想调具体分数或顶刊名单属于高级微调，告诉开发者即可。

      🔎 检索引擎（嵌入 / 重排）

      检索用的向量引擎。填一个 SiliconFlow 免费 Key，检索、对话、深索摘要、文件夹抽题录都共用它——一次配好、全局通用。⚠️ 已建过库后更换引擎（本地↔API 或换模型），旧向量不兼容，点「应用检索引擎」会自动重建。

      👉 还没有 Key？去 SiliconFlow（硅基流动） 登录即领免费额度。填这一个 Key 即可全局复用。
